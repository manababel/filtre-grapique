Convolution Mathématique Avancée

Documentation - Algorithme GaussianBlur_Conv

Flou gaussien mathématique de haute précision par convolution séparable 1D

1. Pourquoi utiliser cette implémentation Gaussienne ?

  • Fidélité Mathématique Absolue : Contrairement aux approximations graphiques rapides (comme l'IIR ou le StackBlur), ce filtre génère le véritable Flou Gaussien d'une précision photographique irréprochable. Le noyau (*kernel*) de convolution est calculé dynamiquement au démarrage à l'aide de l'authentique fonction exponentielle de la cloche de Gauss.
  • Séparabilité Spatiale 2D en 1D : Effectuer une matrice de convolution 2D classique (un gros carré) de rayon $R$ requiert $(2R+1) \times (2R+1)$ lectures par pixel. Grâce à la séparabilité de la fonction Gaussienne, votre code divise drastiquement ce coût : il fait d'abord une **passe 1D horizontale**, stocke le résultat dans un tampon temporaire (`*tempo`), puis enchaîne sur une **passe 1D verticale**.

2. Les Inconvénients et Limites du Filtre

  • Complexité dépendante du rayon — $O(N \times R)$ : C'est son principal point faible par rapport aux filtres de type SAT (Summed-Area Table), IIR ou StackBlur dont le temps d'exécution reste parfaitement constant. Ici, plus le rayon ($R$) augmente, plus le nombre de lectures en mémoire par pixel augmente linéairement. Le filtre devient lourd sur de très grands rayons.
  • Surenchère d'allocations de mémoire vive : La technique de séparabilité exige la création d'un calque tampon intermédiaire (`*tempo`) de taille strictement identique à l'image d'origine. Sur de très grandes textures (images 4K ou 8K), cela engendre une consommation de RAM non négligeable.
  • Coût mathématique à l'initialisation : La génération dynamique du noyau fait appel à la fonction exponentielle (`Exp()`), particulièrement gourmande en cycles CPU. Bien que ce calcul ne soit fait qu'une seule fois en début de traitement, il ajoute une surcharge d'évaluation.

3. Structure et Intégration (PureBasic)

La Fonction d'Appel Publique

Cette API standardise l'entrée utilisateur en assignant le rayon sélectionné dans la pile de paramètres globaux du moteur avant d'exécuter la fonction maîtresse.

Procedure GaussianBlur_Conv(source, cible, mask, rayon)
  Set_Source(source)
  Set_Cible(cible)
  Set_Mask(mask)
  With FilterCtx
    \option[0] = rayon
  EndWith
  GaussianBlur_ConvEx(FilterCtx.FilterParams)
EndProcedure

Le Noyau Directeur : GaussianBlur_ConvEx

C'est l'orchestrateur principal. Il s'occupe de valider les images, d'allouer la mémoire de travail, de générer le vecteur de floats représentant la courbe de Gauss (sigma = rayon / 2), de le **normaliser** pour éviter que l'image ne s'assombrisse ou ne s'illumine, puis de lancer les traitements séquentiels.

; Extrait du traitement séquentiel interne :
Protected sigma.f = radius * 0.5
Protected invTwoSigmaSq.f = 1.0 / (2.0 * sigma * sigma)

; Génération des poids du filtre (Courbe de Gauss)
For i = 0 To size - 1
  x = i - radius
  var = Exp(-x * x * invTwoSigmaSq)
  PokeF(*kernel + (i * SizeOf(Float)), var)
  sum + var
Next

; Normalisation : La somme des coefficients doit être strictement égale à 1.0
Protected invSum.f = 1.0 / sum
For i = 0 To size - 1
  PokeF(*kernel + i * 4, PeekF(*kernel + i * 4) * invSum)
Next

4. Le Filtre : Comment ça marche ?

Le traitement divise le problème bidimensionnel en deux sous-étapes indépendantes exécutées l'une après l'autre.

L'astuce de la Normalisation

Le calcul d'une exponentielle génère des valeurs décimales brutes. Si l'on additionnait ces valeurs directement sur nos couleurs, l'image deviendrait complètement blanche ou noire. La boucle de **normalisation** divise chaque valeur par la somme totale du tableau. Ainsi, le mélange conserve rigoureusement la luminosité d'origine de la scène.

La Révolution de la Séparabilité Spatiale

Votre code applique un traitement orthogonal croisé redoutablement efficace :

Formule d'une cellule de la cloche (1D) :

$$G(x) = e^{-\frac{x^2}{2\sigma^2}}$$

Passe 1 (Horizontale) : Le pixel central P fusionne horizontalement avec ses voisins directs pondérés par le noyau, puis écrit le résultat dans le tampon d'échange.
Passe 2 (Verticale) : Le moteur repart du tampon et applique la même logique verticalement pour restituer la distribution 2D finale sur l'image cible.

Gestion des Bords de l'Image (Clamping)

Une attention particulière a été portée dans vos routines pour empêcher le filtre de lire en dehors de la mémoire lorsque la convolution flirte avec les frontières de la texture :

If i < 0 : i = 0 : ElseIf i > wMinus1 : i = wMinus1 : EndIf

Si le voisin théorique sort des limites à gauche ou à droite, l'index est verrouillé (*clamped*) sur le pixel le plus au bord disponible, évitant ainsi un crash de type Invalid Memory Access.

Logique Interne de l'Algorithme (Pseudo-code Global)

Voici la chronologie d'exécution exacte des blocs de votre programme :

Entrée: Image Source, Image Cible, Rayon

// 1. INITIALISATION
Calculer_Vecteur_Gaussien_1D(Rayon, Noyau_Kernel)
Normaliser_Vecteur(Noyau_Kernel)
Allouer_Buffer_Temporaire(*tempo, Largeur, Hauteur)

// 2. PASSE HORIZONTALE
Pour chaque Ligne Y de l'image (de 0 à Hauteur - 1)
    Offset_Ligne = Y * Largeur
    Pour chaque Colonne X (de 0 à Largeur - 1)
        Rouge = 0 : Vert = 0 : Bleu = 0
        
        Pour k = -Rayon To Rayon
            IndexX = X + k
            Ajuster_Index_Bornes_Image(IndexX, 0, Largeur - 1)
            
            Poids = Lire_Float(*kernel, k + Rayon)
            Couleur_Pixel = Image_Source[Offset_Ligne + IndexX]
            
            Rouge + Couleur_Pixel_R * Poids
            Vert  + Couleur_Pixel_V * Poids
            Bleu  + Couleur_Pixel_B * Poids
        Next k
        
        *tempo[Offset_Ligne + X] = Réassembler_ARGB(255, Rouge, Vert, Bleu)
    Next X
Fin Pour

// 3. PASSE VERTICALE
Pour chaque Ligne Y de l'image (de 0 à Hauteur - 1)
    Pour chaque Colonne X (de 0 à Largeur - 1)
        Rouge = 0 : Vert = 0 : Bleu = 0
        
        Pour k = -Rayon To Rayon
            IndexY = Y + k
            Ajuster_Index_Bornes_Image(IndexY, 0, Hauteur - 1)
            
            Poids = Lire_Float(*kernel, k + Rayon)
            Couleur_Pixel = *tempo[IndexY * Largeur + X] // Lecture du tampon horizontal
            
            Rouge + Couleur_Pixel_R * Poids
            Vert  + Couleur_Pixel_V * Poids
            Bleu  + Couleur_Pixel_B * Poids
        Next k
        
        Image_Cible[Y * Largeur + X] = Réassembler_ARGB(255, Rouge, Vert, Bleu)
    Next X
Fin Pour

// 4. NETTOYAGE
Libérer_Mémoire(*tempo)
Libérer_Mémoire(*kernel)
Fin
