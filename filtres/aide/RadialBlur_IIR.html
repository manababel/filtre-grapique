Filtrage Récurrent Exponentiel

Documentation - Algorithme RadialBlur_IIR

Flou radial infini par cartographie polaire et lissage récursif à mémoire de signal

1. Pourquoi utiliser ce filtre Radial IIR ?

  • Flou d'une portée infinie à coût constant : L'utilisation d'un filtre IIR (Réponse Impulsionnelle Infinie) permet de propager le flou de manière continue le long du rayon. Chaque pixel traité devient la "mémoire" du filtre pour le pixel suivant. L'atténuation exponentielle simule un flou immense sans nécessiter de boucles imbriquées lourdes dépendant du rayon.
  • LookUp Tables (LUT) pour la trigonométrie : Calculer des fonctions Cos() et Sin() au cœur d'un balayage angulaire est extrêmement pénalisant. Votre code génère dynamiquement les tableaux de coordonnées circulaires rc.f() et rs.f() au démarrage. La boucle principale se contente de lire les pointeurs pré-calculés via l'adresse mémoire (décalage de bits i << 2).
  • Résolution angulaire modulable (Qualité) : L'algorithme ne se limite pas à un balayage pas à pas sur 360 degrés. Le coefficient de qualité multiplie la densité des rayons projetés (360 * qualité), ce qui offre un contrôle fin sur la netteté et la granularité périphérique de l'effet "zoom".

2. Les Inconvénients et Limites du Filtre

  • Effet de moirage et "Rayons de bicyclette" : Comme le balayage s'effectue de manière polaire (du centre vers l'extérieur), l'écartement spatial entre deux rayons de flou augmente à mesure qu'on s'éloigne du centre (cx, cy). En périphérie, des lignes d'aliasing distinctes apparaissent, créant un motif en forme d'étoile ou de roues à rayons si le paramètre de qualité est trop bas.
  • Artefacts d'échantillonnage par plus proche voisin (Nearest Neighbor) : La conversion des coordonnées flottantes de la ligne en positions mémoires entières (px, py) se fait par troncature brute. L'absence d'interpolation bilinéaire engendre de petites saccades visuelles le long de la course du flou.
  • Écritures concurrentes et écrasement sur le foyer : Tous les rayons partent obligatoirement du même point central (cx, cy) pour s'étendre vers l'extérieur. Le pixel du centre (et ses voisins immédiats) est donc relu, recalculé et réécrit des milliers de fois consécutives au cours du même traitement, ce qui s'avère lourd et algorithmiquement redondant.

3. Structure et Intégration (PureBasic)

La Fonction d'Appel Publique

L'interface utilisateur configure le rayon d'atténuation (Alpha), le positionnement focal du point d'impact en pourcentage, ainsi que le multiplicateur de précision angulaire.

Procedure RadialBlur_IIR(source, cible, mask, Rayon, posx, posy, qualite)
  Set_Source(source)
  Set_Cible(cible)
  Set_Mask(mask)
  With FilterCtx
    \option[0] = Rayon
    \option[1] = posx
    \option[2] = posy
    \option[3] = qualite
  EndWith
  RadialBlurEx(FilterCtx.FilterParams)
EndProcedure

Génération de la LookUp Table (LUT)

Cette routine pré-calcule les projections vectorielles sur un cercle complet subdivisé selon le niveau de qualité demandé.

; Extrait de la phase de préparation des tableaux d'angles :
Protected quality = \option[3]
Protected inv_quality.f = 1.0 / quality
Protected Dim rc.f(360 * quality)
Protected Dim rs.f(360 * quality)

For i = 0 To (360 * quality) - 1
  angle = Radian(i * inv_quality) 
  rc(i) = Cos(angle) ; Stockage du pas horizontal
  rs(i) = Sin(angle) ; Stockage du pas vertical
Next
\addr[2] = @rc()
\addr[3] = @rs()

4. Le Filtre : Comment ça marche ?

Le traitement abandonne le balayage classique par lignes/colonnes. Il projette des lignes de pixels rayonnant depuis le centre vers les bords extérieurs de l'image.

Le coefficient de rétroaction exponentielle (Alpha)

La force du flou est régie par l'équation d'atténuation temporelle : Alpha = Exp(-2.3 / (Rayon + 1)).
À chaque pas j le long du rayon, la nouvelle couleur est une fusion pondérée de l'état précédent de la ligne (Alpha * r) et de la couleur brute du pixel actuel ((1 - Alpha) * r1).

L'équation de récursivité IIR :

Facteur_Inverse = 1.0 - Alpha

Accumulateur = Alpha * Accumulateur + Facteur_Inverse * Pixel_Courant

Le premier pixel rencontré (j = 0) initialise l'accumulateur (firstPixel = True). Au pas suivant, l'accumulateur absorbe une fraction du nouveau pixel. Plus on s'éloigne du centre, plus les couleurs traversées précédemment s'estompent et se mélangent de façon exponentielle.

Sécurité anti-débordement de la ligne de visée

Puisque les rayons s'étendent jusqu'à la diagonale maximale de l'image (maxRadius), ils finissent invariablement par sortir du cadre. Le filtre intègre une condition d'arrêt et de protection :

If px < 0 Or py < 0 Or px >= lg Or py >= ht : Continue : EndIf

L'instruction Continue permet d'ignorer instantanément les calculs en sautant au point d'échantillonnage suivant de la boucle sans interrompre la rotation globale du filtre.

Logique Interne de l'Algorithme (Pseudo-code Synchrone)

Voici l'enchaînement logique complet du traitement par balayage de rayons :

Entrée: Image Source, Image Cible, Paramètre Rayon, Centre X (%), Centre Y (%), Qualité

// 1. PHASE DES TABLES ET COEFFICIENTS
Foyer_X = (Centre_X * Largeur_Image) / 100
Foyer_Y = (Centre_Y * Hauteur_Image) / 100
Diagonale_Maximale = Sqr(Largeur_Image * Largeur_Image + Hauteur_Image * Hauteur_Image)

Alpha = Exp(-2.3 / (Rayon + 1))
Inverse_Alpha = 1.0 - Alpha
Total_Rayons = 360 * Qualité

Générer_Tables_Trigonométriques_LUT(Table_Cos, Table_Sin, Total_Rayons)

// 2. BALAYAGE ANGULAIRE (Repère Polaire)
Pour chaque_rayon i de 0 à Total_Rayons - 1
    Pas_Cos = Table_Cos[i]
    Pas_Sin = Table_Sin[i]
    
    Accumulateur_R = 0 : Accumulateur_G = 0 : Accumulateur_B = 0
    Drapeau_Premier_Pixel = True
    
    // Progression le long de la ligne droite du rayon
    Pour j de 0 à Diagonale_Maximale
        Pixel_X = Foyer_X + (j * Pas_Cos)
        Pixel_Y = Foyer_Y + (j * Pas_Sin)
        
        // Validation stricte du confinement dans l'image
        Si Pixel_X < 0 Ou Pixel_Y < 0 Ou Pixel_X >= Largeur_Image Ou Pixel_Y >= Hauteur_Image
            Continuer la boucle j (prochain pas)
        Fin Si
        
        Pos_Memoire = (Int(Pixel_Y) * Largeur_Image + Int(Pixel_X)) * 4
        Pixel_Couleur = Lire_Pixel_Source(Pos_Memoire)
        (A, R, G, B) = Extraire_ARGB(Pixel_Couleur)
        
        Si Drapeau_Premier_Pixel == True
            Accumulateur_R = R
            Accumulateur_G = G
            Accumulateur_B = B
            Drapeau_Premier_Pixel = False
        Sinon
            // Équation de récursivité IIR
            Accumulateur_R = Alpha * Accumulateur_R + Inverse_Alpha * R
            Accumulateur_G = Alpha * Accumulateur_G + Inverse_Alpha * G
            Accumulateur_B = Alpha * Accumulateur_B + Inverse_Alpha * B
        Fin Si
        
        R_Final = Clamping(Accumulateur_R)
        G_Final = Clamping(Accumulateur_G)
        B_Final = Clamping(Accumulateur_B)
        
        Écrire_Pixel_Cible(Pos_Memoire, Réassembler_ARGB(A, R_Final, G_Final, B_Final))
    Next j
