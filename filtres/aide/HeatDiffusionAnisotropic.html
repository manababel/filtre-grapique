Équation Différentielle Partielle (EDP)

Documentation - Algorithme HeatDiffusionAnisoBlur

Flou anisotrope intelligent par conduction thermique sélective (Perona-Malik)

1. Pourquoi utiliser ce filtre anisotrope ?

  • Lissage préservant les contours : Contrairement à un flou gaussien standard qui est "isotrope" (il lisse aveuglément tout sur son passage), ce filtre simule la propagation physique de la chaleur dans un milieu hétérogène. Le flou s'arrête net dès qu'il rencontre une transition brutale de couleur (un contour).
  • Optimisation par Table de Correspondance (Lookup Table) : L'évaluation de l'équation de Perona-Malik requiert une fonction exponentielle par pixel et par voisin. Votre code élimine ce goulot d'étranglement en pré-calculant les 256 coefficients de conductivité possibles au démarrage de la procédure dans la table \addr[3]. Le traitement interne n'effectue plus qu'un simple PeekF ultra-rapide.
  • Stratégie de Double Tampon (Ping-Pong Buffering) : Pour que la diffusion mathématique se propage sainement sans altérer les calculs de voisinage en cours, le filtre utilise deux espaces mémoires de taille identique. À chaque itération, l'algorithme alterne le rôle des pointeurs via l'instruction Swap *buf_src, *buf_dst.

2. Les Inconvénients et Limites du Filtre

  • Surcharge liée au processus itératif : Pour obtenir un résultat visible et un lissage de type "peinture à l'huile", le filtre doit s'exécuter sur plusieurs cycles (variable iterations). Si une image demande 50 itérations, la totalité des pixels de l'image devra être lue, calculée et réécrite 50 fois de suite, rendant l'algorithme lourd.
  • Instabilité de la vitesse de diffusion (Paramètre Lambda) : Le coefficient de vitesse de diffusion lambda (contrôlé par \option[2]) est limité à une valeur maximale stricte. Si la valeur configurée dépasse 0.25 (25%), l'équation mathématique diverge : le filtre devient instable, génère des artéfacts numériques violents et détruit la cohérence visuelle des pixels.
  • Difficulté de calibrage du seuil de contraste (K) : Le paramètre de contraste K (\option[1]) définit la sensibilité du détecteur de contours. Si K est trop bas, aucune diffusion n'a lieu (l'image reste intacte). Si K est trop haut, le filtre considère les contours comme du bruit et se comporte comme un simple flou gaussien classique, annulant tout l'intérêt de l'anisotropie.

3. Structure et Intégration (PureBasic)

La Fonction d'Appel Publique

L'interface initialise les trois options de contrôle : le nombre de passes, le seuil de contraste (K) et la vitesse de diffusion exprimée en pourcentage.

Procedure HeatDiffusionBlur(source, cible, mask, iterations, contraste, lambda_percent)
  Set_Source(source)
  Set_Cible(cible)
  Set_Mask(mask)
  With FilterCtx
    \option[0] = iterations
    \option[1] = contraste
    \option[2] = lambda_percent
  EndWith
  HeatDiffusionBlurEx(FilterCtx.FilterParams)
EndProcedure

Le Moteur du Noyau : Génération de la table Exponentielle

Le programme pré-calcule la courbe d'atténuation. Plus l'écart de couleur entre deux pixels est grand, plus la conductivité tend vers 0.

; Extrait de la phase d'initialisation :
Protected invK.f = 1.0 / \option[1]
Protected invKSq.f = invK * invK

For i = 0 To 255
  ; Formule de Perona-Malik : c(g) = Exp(-(g/K)²)
  var = Exp(-i * i * invKSq)
  PokeF(\addr[3] + (i << 2), var)
Next

4. Le Filtre : Comment ça marche ?

À chaque itération, le pixel central analyse ses 4 voisins directs (Nord, Sud, Ouest, Est) pour mesurer le gradient thermique local.

Le Rôle de la Macro d'Évolution (HeatDiffusionAnisoBlur_sp1)

La macro calcule l'écart absolu de couleur entre le pixel courant et ses voisins. Cet écart sert d'index dans la table pour obtenir les coefficients d'atténuation cN, cS, cW, cE. Le pixel central absorbe ou transmet ensuite une fraction de couleur régulée par le paramètre lambda.

Le Voisinage en Croix de Von Neumann

Voisins Analysés
Nord
OuestPixelEst
Sud

L'équation d'ajustement de la valeur :

Variation = cN * (Nord - Pixel) + cS * (Sud - Pixel) + cW * (Ouest - Pixel) + cE * (Est - Pixel)

Nouvelle_Valeur = Pixel + Lambda * Variation

Si la différence de couleur entre le pixel central et son voisin du Nord est inférieure au seuil K, le coefficient cN sera proche de 1.0, permettant l'homogénéisation. Si l'écart est supérieur à K, cN chute vers 0.0 : la barrière est fermée, le contour est préservé.

Gestion des Frontières de l'Image

Le code intègre un système de repli automatique en cas de contact avec les bordures de la matrice :

If y > 0 : getrgb(*source\pixel[pos - lg], Nr...) : Else : Nr = r : EndIf

Lorsqu'un pixel se trouve sur la ligne tout en haut de l'image (Y = 0), il n'a pas de voisin Nord. L'algorithme clone alors de manière transparente les valeurs du pixel central lui-même (Nr = r), ce qui annule proprement le transfert thermique vers l'extérieur.

Logique Interne du Système (Pseudo-code Global)

Voici la séquence séquentielle exacte du traitement de diffusion :

Entrée: Image Source, Image Cible, Itérations, Contraste K, Vitesse Lambda

// 1. PHASE D'INITIALISATION
Lambda_Réel = Vitesse_Lambda * 0.01
Allouer_Table_Correspondance(256 cases * 4 octets)
Pour i de 0 à 255
    Table[i] = Exp(-i * i / (K * K))
Next i

Allouer_Buffer_Temporaire(*tempo, Taille_Image)
Copier_Mémoire(Image_Source -> *tempo)

Pointeur_Source = *tempo
Pointeur_Cible = Image_Cible

// 2. BOUCLE PRINCIPALE DES ITÉRATIONS (Cycle de diffusion)
Pour cycle = 1 To Itérations
    
    // Pour chaque pixel de la matrice
    Pour chaque Ligne Y de 0 à Hauteur - 1
        Pour chaque Colonne X de 0 à Largeur - 1
            Pos = Y * Largeur + X
            (R, G, B) = Extraire_RGB(Pointeur_Source[Pos])
            
            // Échantillonnage avec sécurité des bords
            (Nr, Ng, Nb) = (Y > 0) ? Extraire_RGB(Pointeur_Source[Pos - Largeur]) : (R, G, B)
            (Sr, Sg, Sb) = (Y < Hauteur - 1) ? Extraire_RGB(Pointeur_Source[Pos + Largeur]) : (R, G, B)
            (Wr, Wg, Wb) = (X > 0) ? Extraire_RGB(Pointeur_Source[Pos - 1]) : (R, G, B)
            (Er, Eg, Eb) = (X < Largeur - 1) ? Extraire_RGB(Pointeur_Source[Pos + 1]) : (R, G, B)
            
            // Application du lissage sélectif
            R = R + Lambda_Réel * (Table[Abs(Nr - R)] * (Nr - R) + Table[Abs(Sr - R)] * (Sr - R) + ...)
            G = G + Lambda_Réel * (Table[Abs(Ng - G)] * (Ng - G) + Table[Abs(Sg - G)] * (Sg - G) + ...)
            B = B + Lambda_Réel * (Table[Abs(Nb - B)] * (Nb - B) + Table[Abs(Sb - B)] * (Sb - B) + ...)
            
            Pointeur_Cible[Pos] = Réassembler_ARGB(255, Clamping(R), Clamping(G), Clamping(B))
        Next X
    Next Y
    
    // Inversion de la source et du buffer pour la passe suivante
    Swap Pointeur_Source, Pointeur_Cible
Next cycle

// 3. ÉTAPE DE VÉRIFICATION FINALE ET NETTOYAGE
Si Pointeur_Source n'est pas le calque final attendu
    Copier_Mémoire(Pointeur_Source -> Image_Cible)
Fin Si

Libérer_Mémoire(*tempo)
Libérer_Mémoire(Table)
Fin
