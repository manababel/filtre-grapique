Échantillonnage Discret Type Shader

Documentation - Algorithme SpiralBlur_Stochastic

Approche de type Monte-Carlo par distribution de poids gaussien et dispersion angulaire adaptative

1. Pourquoi utiliser cette implémentation Stochastique ?

  • Vitesse d'Exécution Révolutionnaire (Gain 10x à 20x) : Au lieu de parcourir l'intégralité des pixels situés le long de la ligne spirale (ce qui demande des centaines de lectures), l'algorithme bride le nombre de prélèvements à un maximum strict (Min(samples, Radius, 16)). Cette économie drastique de boucles simule le comportement ultra-rapide des processeurs graphiques (GPU).
  • Distribution Gaussienne Intégrée : L'atténuation du flou n'est pas linéaire. Chaque échantillon reçoit un poids basé sur une approximation gaussienne clochée (Exp(-(offset*offset) / (2*Radius*Radius))). Les points spatialement proches de la position originale pèsent plus lourd que les points périphériques, offrant un lissage de transition de haute facture.
  • Modulation Angulaire par Distance : Le déplacement hélicoïdal s'adapte en temps réel à l'éloignement par rapport au centre. La variable de rotation (force * dist * direction * s * invSamples / 100.0) utilise la distance (dist) comme multiplicateur. Plus le pixel analysé s'éloigne du centre, plus l'arc de la spirale s'étire et s'enroule de façon dynamique.

2. Les Inconvénients et Limites du Filtre

  • Grain de dispersion (Bruit stochastique) : En limitant le balayage à 16 échantillons seulement pour de grands rayons, l'algorithme effectue des sauts spatiaux importants. Cela engendre une perte de précision locale et produit un aspect légèrement granuleux ou haché (aliasing de sous-échantillonnage), typique des rendus Monte-Carlo sans post-débruitage.
  • Surcharge Trigonométrique Redondante : Contrairement à d'autres versions optimisées, le code appelle les fonctions Cos() et Sin() à l'intérieur de la sous-boucle pour chaque échantillon de chaque pixel. L'absence d'utilisation des tables de sinus/cosinus pourtant précalculées dans la fonction d'initialisation limite le potentiel maximal de vitesse du filtre.
  • Incohérence du code de liaison (Bug d'appel) : La procédure d'appel publique spiral_stochastic() redirige par erreur son exécution vers l'ancien moteur récursif SpiralBlur_IIREx au lieu de cibler son propre interpréteur spiral_stochasticEx. Cette anomalie matérielle neutralise l'appel stochastique en l'état.

4. Le Filtre : Comment ça marche ?

Pour chaque pixel de la texture, l'algorithme détermine l'angle d'origine, puis applique une déviation combinant décalage linéaire (offset) et rotation proportionnelle.

Calcul du Rayon et de l'Angle Propre

L'algorithme bascule chaque pixel dans un repère polaire virtuel centré sur l'axe pivot :
dist = Sqr(dx*dx + dy*dy)
baseAngle = ATan2(dy, dx)
La spirale se déploie ensuite en ajoutant la force de torsion à cet angle de base.

Formule de Dispersion des Points :

Offset_Gaussien = (s - Bornes/2) * (Rayon / Bornes)
Angle_Spirale = Angle_Base + Rotation_Proportionnelle

Nouveau_Rayon = Distance_Centre + Offset_Gaussien

Plutôt que d'analyser chaque pixel contigu, l'algorithme "saute" d'un point à un autre le long de la trajectoire hélicoïdale. Chaque pixel prélevé injecte sa couleur dans l'accumulateur au prorata de son coefficient d'atténuation weight.

Logique Interne de l'Algorithme (Pseudo-code Synchrone)

Voici la retranscription structurelle des calculs de rendu stochastique par pixel :

Entrée: Image Source, Image Cible, Rayon, Foyer X (%), Foyer Y (%), Force, Sens

// 1. CALCULS DES COORDONNÉES DU FOYER EN PIXELS
Axe_X = Largeur_Image * (Foyer_X / 100.0)
Axe_Y = Hauteur_Image * (Foyer_Y / 100.0)

Incrément_Direction = (Sens * 2) - 1

// Détermination du nombre maximal de prélèvements (Maximum 16)
Nombre_Echantillons = Fixer_Minimum(Rayon, 16)
Inverse_Echantillons = 1.0 / Nombre_Echantillons

// 2. BALAYAGE DE LA SURFACE DE L'IMAGE
Pour chaque Ligne Y de 0 à Hauteur - 1
    Pour chaque Colonne X de 0 à Largeur - 1
        
        Distance_X = X - Axe_X
        Distance_Y = Y - Axe_Y
        Distance_Centre = Sqr(Distance_X * Distance_X + Distance_Y * Distance_Y)
        
        Angle_Polaire_Base = ATan2(Distance_Y, Distance_X)
        
        Cumul_R = 0.0 : Cumul_G = 0.0 : Cumul_B = 0.0
        Somme_Poids_Total = 0.0 : Alpha_Sortie = 255
        
        // 3. BOUCLE DE BALAYAGE STOCHASTIQUE (STYLE SHADER COUPE MONTE-CARLO)
        Pour s = 0 To Nombre_Echantillons - 1
            
            // Écartement linéaire par rapport au rayon
            Offset_Spatiale = (s - Nombre_Echantillons / 2.0) * (Rayon / (Nombre_Echantillons / 2.0))
            
            // Évaluation du filtre de profil gaussien
            Poids_Gauss = Exp(-(Offset_Spatiale * Offset_Spatiale) / (2.0 * Rayon * Rayon))
            
            // Calcul de la déviation de l'angle de la spirale
            Facteur_Rotation = (Force * Distance_Centre * Incrément_Direction * s * Inverse_Echantillons) / 100.0
            Angle_Echantillon = Angle_Polaire_Base + Convertir_En_Radians(Facteur_Rotation)
            
            Rayon_Echantillon = Distance_Centre + Offset_Spatiale
            
            // Projection cartésienne finale de l'échantillon
            Proj_X = Axe_X + Rayon_Echantillon * Cos(Angle_Echantillon)
            Proj_Y = Axe_Y + Rayon_Echantillon * Sin(Angle_Echantillon)
            
            Int_X = Convertir_En_Entier(Proj_X)
            Int_Y = Convertir_En_Entier(Proj_Y)
            
            // Test de confinement et d'accès direct à la mémoire
            Si Int_X >= 0 Et Int_X < Largeur_Image Et Int_Y >= 0 Et Int_Y < Hauteur_Image
                Pixel = Lire_Pixel_Source(Int_X, Int_Y)
                
                Cumul_R + Poids_Gauss * Extraire_Rouge(Pixel)
                Cumul_G + Poids_Gauss * Extraire_Vert(Pixel)
                Cumul_B + Poids_Gauss * Extraire_Bleu(Pixel)
                Alpha_Sortie = Extraire_Alpha(Pixel)
                
                Somme_Poids_Total + Poids_Gauss
            Fin Si
        Next s
        
        // 4. NORMALISATION ET ÉCRITURE DU PIXEL
        Si Somme_Poids_Total > 0 Alors
            R_Final = Int(Cumul_R / Somme_Poids_Total)
            G_Final = Int(Cumul_G / Somme_Poids_Total)
            B_Final = Int(Cumul_B / Somme_Poids_Total)
            
            Écrire_Pixel_Cible(X, Y, Réassembler_ARGB(Alpha_Sortie, R_Final, G_Final, B_Final))
        Fin Si
        
    Next X
Fin Pour
