Échantillonnage Angulaire Symétrique

Documentation - Algorithme RotationalBlur

Effet de filé cinétique circulaire par rotation matricielle vectorielle et sommation fenêtrée

1. Pourquoi utiliser cette implémentation de Flou de Rotation ?

  • Distribution Équilibrée du Flou (Centrage de l'Arc) : Plutôt que d'effectuer une rotation brute allant de 0 à l'Angle cible, votre code répartit symétriquement l'échantillonnage de part et d'autre de la position initiale du pixel (-angle * 0.5 + angle * t). L'image reste ainsi parfaitement calée sur ses formes d'origine sans subir de décalage ou de torsion unilatérale.
  • Correction Dynamique des Bords (Compteur Flottant) : Lorsque les coordonnées calculées le long de l'arc de cercle sortent des frontières de la texture, l'algorithme rejette proprement le point (Continue). La moyenne finale s'ajuste dynamiquement en divisant uniquement par le nombre de points valides effectivement collectés (variable count), évitant l'apparition de franges noires ou transparentes en bordure.
  • Préservation de la netteté en cas de rejet total : Si la trajectoire circulaire d'un pixel se situe intégralement en dehors du cadre de l'image (count = 0), le filtre bascule automatiquement sur une sécurité (bloc Else) qui réinjecte le pixel source d'origine sans altération, éliminant tout risque d'affichage de pixel mort ou vide.

2. Les Inconvénients et Limites du Filtre

  • Effet de crénelage en hélice (Aliasing) : La distance spatiale parcourue par l'arc de rotation augmente proportionnellement à l'éloignement par rapport au point central. À l'extrême périphérie de l'image, le nombre maximal de 50 échantillons devient insuffisant, transformant le flou en une suite de points pointillés ou de motifs hachés disgracieux.
  • Lenteur critique liée aux calculs trigonométriques : Contrairement à la version IIR par rayonnement, ce filtre recalcule les fonctions Cos(rotAngle) et Sin(rotAngle) à chaque pas de la sous-boucle pour chaque pixel de l'image. L'absence de table de correspondance (LUT) rend l'algorithme particulièrement gourmand en temps processeur.
  • Inégalité du flou selon la distance (Gradient Spatial) : Près du centre (cx, cy), les vecteurs de déplacement (dx, dy) sont minuscules, le flou y est donc pratiquement invisible. À l'inverse, les zones éloignées tournent à grande vitesse. Cette spécificité mathématique interdit l'obtention d'un flou circulaire uniforme sur l'intégralité du calque.

3. Structure et Intégration (PureBasic)

La Fonction d'Appel Publique

L'interface convertit les positions en valeurs pixelisées et reçoit l'amplitude angulaire du mouvement ainsi que la densité de balayage.

Procedure RotationalBlur(source, cible, mask, cx, cy, angle, echantillons)
  Set_Source(source)
  Set_Cible(cible)
  Set_Mask(mask)
  With FilterCtx
    \option[0] = cx
    \option[1] = cy
    \option[2] = angle
    \option[3] = echantillons
  EndWith
  RotationalBlurEx(FilterCtx.FilterParams)
EndProcedure

Détermination du Centrage Angulaire

Cette section montre comment l'algorithme calcule le décalage de phase pour ancrer l'échantillonnage de façon équitable autour de la coordonnée de base.

; Conversion de l'angle d'entrée en radians :
Protected angle.f = \option[2] * #PI / 180.0

; Dans la sous-boucle (i évoluant de 0 à samples - 1) :
Protected t.f = i / (samples - 1.0)
rotAngle = -angle * 0.5 + angle * t ; Balayage de -DemiAngle à +DemiAngle

4. Le Filtre : Comment ça marche ?

Pour chaque coordonnée spatiale, l'algorithme extrait le rayon qui la relie au centre, puis applique une matrice de rotation trigonométrique standard bidimensionnelle.

La Matrice de Rotation Vectorielle

La déviation d'un point autour du repère focal s'obtient en croisant les coordonnées de distance relative avec les rapports sinus/cosinus de l'angle courant :
rx = dx * cosA - dy * sinA
ry = dx * sinA + dy * cosA
Les nouvelles coordonnées absolues de lecture deviennent alors : sx = cx + rx et sy = cy + ry.

Formule de Calcul Spatiale :

Distance_X = X_Courant - Centre_X
Distance_Y = Y_Courant - Centre_Y

Nouveau_X = Centre_X + (Distance_X * Cos - Distance_Y * Sin)

L'indice de progression t pilote la rotation. À chaque étape, la position du pixel d'origine subit une légère déviation circulaire. L'ensemble des points d'échantillonnage dessine un arc de cercle parfait dont la courbure dépend de la distance par rapport au Centre focal.

Sécurité par Contrôle de Limites Discrètes

La vérification des coordonnées exclut immédiatement les calculs d'extraction colorimétrique si le pixel ciblé se trouve hors-champ de la mémoire vidéo :

If sx < 0 Or sx >= lg Or sy < 0 Or sy >= ht : Continue : EndIf

L'utilisation du mot-clé Int() lors de la conversion finale de l'adresse ((Int(sy) * lg + Int(sx)) << 2) réalise une interpolation par troncature brute (plus proche voisin), garantissant un pointage mémoire rapide et intègre.

Logique Interne de l'Algorithme (Pseudo-code Synchrone)

Voici l'ordonnancement séquentiel des instructions du filtre de flou rotationnel :

Entrée: Image Source, Image Cible, Centre X (%), Centre Y (%), Angle (Degrés), Échantillons

// 1. CONVERSION ET VERROUILLAGE DES BORNES
Si Échantillons < 2 Alors Échantillons = 2
Si Échantillons > 50 Alors Échantillons = 50

Foyer_X = Largeur_Image * (Centre_X / 100.0)
Foyer_Y = Hauteur_Image * (Centre_Y / 100.0)
Angle_Rad = Angle * PI / 180.0

// 2. BALAYAGE DE LA SURFACE
Pour chaque Ligne Y de 0 à Hauteur - 1
    Pour chaque Colonne X de 0 à Largeur - 1
        
        Somme_A = 0.0 : Somme_R = 0.0 : Somme_G = 0.0 : Somme_B = 0.0
        Points_Valides = 0
        
        // Coordonnées de distance par rapport à l'axe
        Delta_X = X - Foyer_X
        Delta_Y = Y - Foyer_Y
        
        // Exploration de l'arc de rotation
        Pour i = 0 To Échantillons - 1
            Ratio_T = i / (Échantillons - 1.0)
            Angle_Etape = -Angle_Rad * 0.5 + Angle_Rad * Ratio_T
            
            Cos_A = Cos(Angle_Etape)
            Sin_A = Sin(Angle_Etape)
            
            // Transformation géométrique matricielle
            Rotation_X = Foyer_X + (Delta_X * Cos_A - Delta_Y * Sin_A)
            Rotation_Y = Foyer_Y + (Delta_X * Sin_A + Delta_Y * Cos_A)
            
            // Test d'exclusion des débordements
            Si Rotation_X >= 0 Et Rotation_X < Largeur_Image Et Rotation_Y >= 0 Et Rotation_Y < Hauteur_Image
                Index_Memoire = (Int(Rotation_Y) * Largeur_Image + Int(Rotation_X)) * 4
                Pixel = Lire_Pixel_Source(Index_Memoire)
                
                Somme_A + Extraire_Alpha(Pixel)
                Somme_R + Extraire_Rouge(Pixel)
                Somme_G + Extraire_Vert(Pixel)
                Somme_B + Extraire_Bleu(Pixel)
                Points_Valides + 1
            Fin Si
        Next i
        
        // 3. ETAPE DE NORMALISATION DES CANAUX
        Si Points_Valides > 0
            Alpha_Final = Somme_A / Points_Valides
            Rouge_Final = Somme_R / Points_Valides
            Vert_Final  = Somme_G / Points_Valides
            Bleu_Final  = Somme_B / Points_Valides
        Sinon
            // Repli de sécurité : restauration du pixel d'origine
            Pixel_Origine = Lire_Pixel_Source((Y * Largeur_Image + X) * 4)
            (Alpha_Final, Rouge_Final, Vert_Final, Bleu_Final) = Extraire_ARGB(Pixel_Origine)
        Fin Si
        
        // Protection contre le dépassement d'octets et écriture
        Pixel_Assemble = Réassembler_ARGB(Clamp(Alpha_Final), Clamp(Rouge_Final), Clamp(Vert_Final), Clamp(Bleu_Final))
        Écrire_Pixel_Cible(X, Y, Pixel_Assemble)
        
    Next X
Fin Pour
