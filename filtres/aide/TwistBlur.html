Échantillonnage par Arc de Torsion

Documentation - Algorithme TwistBlur

Effet de tourbillonnement localisé par atténuation géométrique linéaire et interpolation circulaire symétrique

1. Pourquoi utiliser cette implémentation de Twist Blur ?

  • Torsion Dégressive Proportionnelle : Contrairement à un flou radial ou directionnel uniforme, cet algorithme calcule un coefficient de pénétration (angleAmount = 1.0 - (distance / radius)). La force de rotation est maximale au centre précis de l'effet, puis décroît linéairement pour s'annuler complètement dès que la distance dépasse la frontière du rayon. Cela permet une intégration parfaite et invisible du flou dans le reste de l'image.
  • Échantillonnage Bidirectionnel Équilibré : L'exploration des pixels ne se fait pas de manière unidirectionnelle. La variable d'arc rotAngle distribue les prélèvements de façon symétrique de part et d'autre de la position d'origine ((t - 0.5) * 2.0, ce qui donne un intervalle allant de -1 à +1). L'image finale ne subit ainsi aucun décentrage visuel.
  • Préservation Native de la Texture : Si un pixel se trouve en dehors de la zone d'influence du rayon, ou si aucun échantillon n'est validé, le filtre bascule automatiquement sur un mode de secours (fallback). Il effectue une lecture directe par PeekL pour recopier fidèlement le pixel d'origine, évitant les zones noires ou les artefacts de calcul.
  • Sécurisation par Bornage (Clamping) : Avant l'écriture finale en mémoire, chaque canal de couleur (A, R, G, B) passe par une fonction de bridage stricte (Clamp(valeur, 0, 255)), ce qui élimine tout risque d'artefact de débordement de type "integer overflow".

2. Les Inconvénients et Limites du Filtre

  • Lenteur Trigonométrique Quadratique : C'est le point faible de cette implémentation. Pour chaque pixel situé à l'intérieur du disque, et pour chaque échantillon demandé, le programme appelle de manière synchrone les fonctions mathématiques lourdes Cos() et Sin(). Pour une image standard avec un échantillonnage moyen, le processeur exécute des millions d'opérations trigonométriques à la volée, sans aucune table de précalcul.
  • Saut d'Index de Fin de Ligne : La boucle principale de balayage vertical est configurée avec la borne supérieure thread_stop - 1. Selon l'arrondi du découpage géométrique des lignes de l'image, cette soustraction peut oublier de traiter la toute dernière ligne de pixels de la zone, laissant une bande non filtrée au bas du rendu.

4. Le Filtre : Comment ça marche ?

Pour chaque pixel de l'image, l'algorithme calcule sa distance par rapport au centre choisi. Si le pixel est dans le rayon, un vecteur de rotation oscille comme un pendule pour faire la moyenne des couleurs environnantes sur un arc de cercle.

Calcul de la Matrice de Rotation 2D

Pour faire pivoter le vecteur de position sans repasser par des coordonnées polaires pures, l'algorithme applique une multiplication matricielle standard sur les deltas de position :
rx = dx * cosA - dy * sinA
ry = dx * sinA + dy * cosA
Les nouvelles coordonnées échantillonnées sont immédiatement reconstituées par rapport au centre : (cx + rx, cy + ry).

Cinématique de l'Arc de Torsion

Balayage en Arc
t=0
Pixel
t=1

Formule de Calcul de l'Angle Oscillant :

angleAmount = 1.0 - (Distance / Rayon)
rotAngle = Angle_Max * angleAmount * (t - 0.5) * 2.0

La variable t évolue linéairement entre 0.0 et 1.0. Le facteur (t - 0.5) * 2.0 transforme cette progression pour balayer l'arc de cercle de façon symétrique, en partant de l'angle négatif maximum jusqu'à l'angle positif maximum.

Logique Interne de l'Algorithme (Pseudo-code Synchrone)

Voici la description pas à pas de la logique de calcul de torsion globale :

Entrée: Image Source, Image Cible, Foyer_X (%), Foyer_Y (%), Angle_Max_Degres, Rayon, Nb_Echantillons

// 1. CONVERSION DES UNITES ET CONTRAINTES
Axe_X_Pixel = Largeur_Image * (Foyer_X / 100.0)
Axe_Y_Pixel = Hauteur_Image * (Foyer_Y / 100.0)
Angle_Max_Radians = Angle_Max_Degres * 3.14159265 / 180.0

Si Nb_Echantillons < 2 Alors Nb_Echantillons = 2
Si Nb_Echantillons > 50 Alors Nb_Echantillons = 50
Si Rayon < 1 Alors Rayon = 1

// 2. PARCOURS GÉOMÉTRIQUE DE L'IMAGE
Pour chaque Ligne Y de 0 à Hauteur_Image - 1
    Pour chaque Colonne X de 0 à Largeur_Image - 1
        
        Cumul_R = 0.0 : Cumul_G = 0.0 : Cumul_B = 0.0 : Cumul_A = 0.0
        Nombre_Points_Valides = 0
        
        Delta_X = X - Axe_X_Pixel
        Delta_Y = Y - Axe_Y_Pixel
        Distance_Centre = Sqr(Delta_X * Delta_X + Delta_Y * Delta_Y)
        
        // Détermination du taux de torsion selon la distance
        Si Distance_Centre <= Rayon Alors
            Facteur_Atténuation = 1.0 - (Distance_Centre / Rayon)
        Sinon
            Facteur_Atténuation = 0.0
        Fin Si
        
        // 3. BOUCLE D'ÉCHANTILLONNAGE SUR L'ARC DE CERCLE
        Pour i = 0 À Nb_Echantillons - 1
            
            Pas_Normalisé = i / (Nb_Echantillons - 1.0) // Évolue de 0.0 à 1.0
            Multiplicateur_Symétrique = (Pas_Normalisé - 0.5) * 2.0 // Évolue de -1.0 à +1.0
            
            Angle_Oscillation = Angle_Max_Radians * Facteur_Atténuation * Multiplicateur_Symétrique
            
            Cos_A = Cos(Angle_Oscillation)
            Sin_A = Sin(Angle_Oscillation)
            
            // Rotation trigonométrique du vecteur position
            Rotated_X = Delta_X * Cos_A - Delta_Y * Sin_A
            Rotated_Y = Delta_X * Sin_A + Delta_Y * Cos_A
            
            Cible_X = Axe_X_Pixel + Rotated_X
            Cible_Y = Axe_Y_Pixel + Rotated_Y
            
            // Test de confinement dans les limites de la texture
            Si Cible_X >= 0 Et Cible_X < Largeur_Image Et Cible_Y >= 0 Et Cible_Y < Hauteur_Image Alors
                
                Pixel = Lire_Pixel_Source_Direct(Int(Cible_X), Int(Cible_Y))
                
                Cumul_R = Cumul_R + Extraire_Rouge(Pixel)
                Cumul_G = Cumul_G + Extraire_Vert(Pixel)
                Cumul_B = Cumul_B + Extraire_Bleu(Pixel)
                Cumul_A = Cumul_A + Extraire_Alpha(Pixel)
                Nombre_Points_Valides = Nombre_Points_Valides + 1
                
            Fin Si
        Next i
        
        // 4. MIXAGE ET COMPRESSION DU PIXEL FINAL
        Si Nombre_Points_Valides > 0 Alors
            A_Final = Cumul_A / Nombre_Points_Valides
            R_Final = Cumul_R / Nombre_Points_Valides
            G_Final = Cumul_G / Nombre_Points_Valides
            B_Final = Cumul_B / Nombre_Points_Valides
        Sinon
            // Mode fallback : Récupération du pixel d'origine
            Pixel_Origine = Lire_Pixel_Source_Direct(X, Y)
            A_Final = Extraire_Alpha(Pixel_Origine)
            R_Final = Extraire_Rouge(Pixel_Origine)
            G_Final = Extraire_Vert(Pixel_Origine)
            B_Final = Extraire_Bleu(Pixel_Origine)
        Fin Si
        
        // Limitation stricte des couleurs (0-255)
        A_Final = Limiter(A_Final, 0, 255)
        R_Final = Limiter(R_Final, 0, 255)
        G_Final = Limiter(G_Final, 0, 255)
        B_Final = Limiter(B_Final, 0, 255)
        
        Écrire_Pixel_Cible(X, Y, Réassembler_ARGB(A_Final, R_Final, G_Final, B_Final))
        
    Next X
Fin Pour
