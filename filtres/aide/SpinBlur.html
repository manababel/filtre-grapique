Atténuation et Pondération Clochée

Documentation - Algorithme SpinBlur

Flou rotationnel perfectionné avec décroissance périphérique et fenêtrage gaussien

1. Pourquoi utiliser cette implémentation de Spin Blur ?

  • Centrage Intelligent Automatique : Le filtre propose une double ergonomie. Si l'utilisateur transmet la valeur -1, l'algorithme calcule de manière autonome le centre géométrique exact de l'image (lg / 2.0, ht / 2.0). Sinon, il accepte des coordonnées de foyer personnalisées en pixels absolus.
  • Contrôle de l'Atténuation Énergétique (Falloff) : Contrairement aux flous de rotation basiques qui forcent le même angle de distorsion partout, l'option falloff calcule la distance (dist) séparant le pixel traité du centre. Elle réduit dynamiquement l'angle effectif à mesure qu'on s'éloigne du cœur, empêchant l'image d'être totalement destructurée sur ses bordures extérieures.
  • Pondération Spatiale Gaussienne : L'algorithme intègre un commutateur de profil. En mode Gaussien, au lieu de faire une moyenne uniforme standard, le filtre applique une fonction exponentielle en cloche (Exp(-t * t * 2.0)). Les sous-échantillons situés au centre de l'arc ont plus de poids que ceux situés aux extrémités de la traînée, ce qui génère un flou cinétique beaucoup plus doux, naturel et réaliste.

2. Les Inconvénients et Limites du Filtre

  • Surcharge Trigonométrique Extrême : Cet algorithme calcule un sinus et un cosinus (Cos(angle) et Sin(angle)) pour chaque échantillon de chaque pixel de la texture. Sans table de pré-calcul (LUT), l'accumulation de fonctions transcendantes engendre une charge CPU colossale.
  • Arrondi par Troncature Décentré (Bias de position) : La conversion des coordonnées d'échantillonnage vers les indices entiers de la texture se fait par un décalage suivi d'une conversion brute : Int(nx + 0.5). Bien que cela simule un arrondi correct, l'absence d'interpolation bilinéaire des 4 pixels voisins induit de légères structures en escalier le long de la courbe de rotation.
  • Perte d'Énergie aux Frontières de l'Image : Lorsqu'un arc de cercle floute un pixel proche d'un angle du cadre, les sous-échantillons qui sortent de l'image sont ignorés. Bien que le code compense la moyenne en utilisant la variable total_weight, les zones situées hors-champ ne participent pas au mélange, brisant l'homogénéité du flou sur le pourtour.

3. Structure et Intégration (PureBasic)

La Fonction d'Appel Publique

L'interface transmet le nombre de points d'échantillonnage, l'angle maximal de rotation, l'origine spatiale du foyer ainsi que les deux variables de profil (atténuation et pondération).

Procedure SpinBlur(source, cible, mask, samples, angle, cx, cy, attenuation, ponderation)
  Set_Source(source)
  Set_Cible(cible)
  Set_Mask(mask)
  With FilterCtx
    \option[0] = samples
    \option[1] = angle
    \option[2] = cx
    \option[3] = cy
    \option[4] = attenuation
    \option[5] = ponderation
  EndWith
  SpinBlurEx(FilterCtx.FilterParams)
EndProcedure

Gestion Modulaire du Foyer

Cette phase montre la logique d'interception du code permettant de basculer instantanément entre le repère automatique et les coordonnées utilisateur.

If \option[2] = -1  ; Mode automatique
  cx = lg / 2.0
  cy = ht / 2.0
Else
  cx = \option[2]  ; Mode manuel
  cy = \option[3]
EndIf

4. Le Filtre : Comment ça marche ?

L'algorithme adapte la taille de son arc de balayage pour chaque pixel en calculant sa distance au centre, puis évalue la distribution des poids d'accumulation.

Calcul du Coefficient d'Atténuation (Falloff)

Pour chaque pixel de l'image, sa distance absolue au centre est projetée par rapport au rayon maximal théorique de la texture :
weight = 1.0 - (dist / (Sqr(lg * lg + ht * ht) / 2.0)) * falloff / 100.0.
L'angle de balayage effectif est ensuite calculé via : effective_angle = angle_max * weight.

Cinématique du Profil de Pondération

Courbe Gaussienne
0.1⬅Extrémité
1.0⬅Centre (Pixel)
0.1⬅Extrémité

Calcul des Poids d'Échantillons (Si Pondération = 1) :

t = (k - Échantillons / 2.0) / (Échantillons / 2.0)

Poids_Échantillon = Exp(-t * t * 2.0)

La variable t évolue entre -1.0 et 1.0 le long du balayage de l'arc. La fonction exponentielle génère une courbe en cloche. Les couleurs prélevées aux extrémités de la traînée de rotation reçoivent un impact faible, tandis que le point de départ conserve une influence majeure.

Logique Interne de l'Algorithme (Pseudo-code Synchrone)

Voici le déroulement séquentiel et mathématique des opérations effectuées par le moteur de flou :

Entrée: Image Source, Image Cible, Nombre Échantillons, Angle Maximum, Centre X, Centre Y, Valeur Atténuation, Type Pondération

// 1. INITIALISATION ET ETALONNAGE DU FOYER
Si Centre_X == -1 Alors
    Foyer_X = Largeur_Image / 2.0
    Foyer_Y = Hauteur_Image / 2.0
Sinon
    Foyer_X = Centre_X
    Foyer_Y = Centre_Y
Fin Si

Angle_Max_Rad = Angle_Maximum * PI / 180.0
Rayon_Max_Image = Sqr(Largeur_Image^2 + Hauteur_Image^2) / 2.0

// 2. BALAYAGE BIDIMENSIONNEL
Pour chaque Ligne Y de 0 à Hauteur - 1
    Pour chaque Colonne X de 0 à Largeur - 1
        
        Delta_X = X - Foyer_X
        Delta_Y = Y - Foyer_Y
        Distance_Axe = Sqr(Delta_X * Delta_X + Delta_Y * Delta_Y)
        
        // Étape A : Atténuation angulaire spatiale (Falloff)
        Si Valeur_Atténuation > 0 Alors
            Poids_Distance = 1.0 - (Distance_Axe / Rayon_Max_Image) * (Valeur_Atténuation / 100.0)
            Si Poids_Distance < 0.1 Alors Poids_Distance = 0.1
        Sinon
            Poids_Distance = 1.0
        Fin Si
        
        Angle_Effectif = Angle_Max_Rad * Poids_Distance
        Pas_Angulaire = Angle_Effectif / (Échantillons - 1)
        
        Somme_A = 0 : Somme_R = 0 : Somme_G = 0 : Somme_B = 0
        Cumul_Poids_Total = 0 : Compteur_Points = 0
        
        // Étape B : Échantillonnage sur l'arc circulaire
        Pour k = 0 To Échantillons - 1
            Angle_Courant = -Angle_Effectif / 2.0 + k * Pas_Angulaire
            
            Cos_A = Cos(Angle_Courant)
            Sin_A = Sin(Angle_Courant)
            
            // Transformation de matrice géométrique
            Rotated_X = Foyer_X + (Delta_X * Cos_A - Delta_Y * Sin_A)
            Rotated_Y = Foyer_Y + (Delta_X * Sin_A + Delta_Y * Cos_A)
            
            Int_X = Int(Rotated_X + 0.5)
            Int_Y = Int(Rotated_Y + 0.5)
            
            // Étape C : Contrôle d'intégrité mémoire et profil de poids
            Si Int_X >= 0 Et Int_X < Largeur_Image Et Int_Y >= 0 Et Int_Y < Hauteur_Image
                
                Poids_Echantillon = 1.0
                Si Type_Pondération == 1 Alors
                    Facteur_T = (k - Échantillons / 2.0) / (Échantillons / 2.0)
                    Poids_Echantillon = Exp(-Facteur_T * Facteur_T * 2.0)
                Fin Si
                
                Pixel = Lire_Pixel_Source(Int_X, Int_Y)
                
                Somme_A + Extraire_Alpha(Pixel) * Poids_Echantillon
                Somme_R + Extraire_Rouge(Pixel) * Poids_Echantillon
                Somme_G + Extraire_Vert(Pixel) * Poids_Echantillon
                Somme_B + Extraire_Bleu(Pixel) * Poids_Echantillon
                
                Cumul_Poids_Total + Poids_Echantillon
                Compteur_Points + 1
            Fin Si
        Next k
        
        // Étape D : Normalisation pondérée
        Si Compteur_Points > 0 Et Cumul_Poids_Total > 0 Alors
            A_Final = Somme_A / Cumul_Poids_Total
            R_Final = Somme_R / Cumul_Poids_Total
            G_Final = Somme_G / Cumul_Poids_Total
            B_Final = Somme_B / Cumul_Poids_Total
        Sinon
            Pixel_Defaut = Lire_Pixel_Source(X, Y)
            (A_Final, R_Final, G_Final, B_Final) = Extraire_ARGB(Pixel_Defaut)
        Fin Si
        
        Écrire_Pixel_Cible(X, Y, Réassembler_ARGB(A_Final, R_Final, G_Final, B_Final))
        
    Next X
Fin Pour
