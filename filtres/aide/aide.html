Guide d'Aide Général

Utilisation de la Bibliothèque de Filtres Graphiques sur CPU

Principe de fonctionnement standardisé pour l'ensemble des filtres graphiques.

Pourquoi ce guide général ?

La bibliothèque intègre beaucoup de filtres graphiques différents. Bien que les algorithmes mathématiques internes varient d'un filtre à l'autre (flous, détection de contours, ajustement de couleurs, distorsions, etc.), tous les filtres fonctionnent de façon similaire dans leur structure et leur syntaxe d'appel.

Chaque filtre est systématiquement séparé en deux approches d'implémentation :

  • Option A (Fonction Standard) : Pour des appels directs, rapides, avec passage de paramètres classiques.
  • Option B (Fonction Étendue "Ex") : Pour les structures de rendu complexes, la manipulation de pointeurs de mémoire brute ou les traitements par lots à l'aide d'un contexte persistant (FilterCtx).

🖥️ Fonctionnement du logiciel de démonstration (Le Labo)

Bien que la bibliothèque propose de très nombreux filtres, sachez que la plupart sont présents à titre de pure démonstration technique et ne vous serviront pas tous . C'est pourquoi l'application vous permet de trier vos outils en ajoutant vos filtres préférés directement dans la section "Favoris" pour y accéder plus rapidement.

L'application de démonstration vous permet de manipuler et de combiner jusqu'à trois types d'images différents dans votre espace de travail :

  • L'Image Source (Obligatoire) : C'est la base de votre travail, l'image principale qui subira la majorité des transformations.
  • L'Image de Mixage (Optionnelle) : Cette image n'a qu'un seul but précis : fusionner son propre contenu avec l'image source. Pour l'activer, vous devez appliquer le filtre "Blend" situé dans le menu des filtres sous l'option Filtre \ Blend modes.
  • L'Image de Masque (Optionnelle) : Cette couche sert exclusivement à appliquer un masque de détourage ou de transparence (niveaux de gris) sur votre image source afin de limiter la zone d'effet d'un traitement.

Comment cibler et appliquer un traitement ?
Les filtres de l'application sont polyvalents. Vous pouvez choisir d'appliquer un filtre sur n'importe laquelle de ces trois images en cours. Pour ce faire :

  1. Sélectionnez l'image désirée (Source, Mixage ou Masque) dans l'interface de la démo.
  2. Choisissez votre filtre et configurez ses paramètres.
  3. Cliquez sur le bouton "Appliquer =>" pour valider le traitement et mettre à jour le rendu en mémoire.

💡 Astuce importante pour optimiser vos programmes

Étant donné que la bibliothèque a beaucoup de filtres similaires, inclure la totalité des fichiers peut alourdir inutilement votre exécutable final. Pour alléger la taille de votre programme et lui permettre de compiler beaucoup plus rapidement, vous pouvez faire le tri avant la compilation :

  1. Dans la section "déclaration de module" (DeclareModule / UseModule), supprimez les déclarations des filtres dont vous n'avez pas besoin.
  2. Dans la section des "includefile", localisez les directives XIncludeFile et supprimez les liaisons vers les fichiers des filtres inutilisés.

Exemples d'implémentation générique (PureBasic)

Les codes ci-dessous sont donnés à titre d'exemples théoriques (non fonctionnels). Ils décrivent la logique d'appel à utiliser. Il vous suffit de remplacer NomDuFiltre par le nom réel du traitement désiré.

Option A : Utilisation Directe (Fonction Standard)

Idéal pour appliquer un filtre rapidement en passant les arguments à la suite.

IncludeFile "filtres.pbi"
UseModule filtres

If OpenWindow(0, 0, 0, 800, 600, "Exemple d'Appel Standard...", #PB_Window_SystemMenu | #PB_Window_ScreenCentered)
  
  SourceImage = 1
  CibleImage  = 2
  AucunMasque = 0
  
  ; Définition des paramètres requis (à adapter selon le filtre choisi)
  Parametre_A = 5 
  Parametre_B = 12
  ModeOption  = 0
 
  file$ = OpenFileRequester("Image","","",0)
  If load_image_32(SourceImage , file$) = 0
    End
  Else
    ResizeImage(SourceImage, 800, 600)
  EndIf
  
  CopyImage(SourceImage, CibleImage)

  ; Appel de la fonction : Remplacer "NomDuFiltre" par le filtre souhaité
  NomDuFiltre(SourceImage, CibleImage, AucunMasque, Parametre_A, Parametre_B, ModeOption)
  
  ; Note : L'image cible peut être identique à la source pour modifier l'original en mémoire :
  ; NomDuFiltre(SourceImage, SourceImage, AucunMasque, Parametre_A, Parametre_B, ModeOption)
  
  StartDrawing(WindowOutput(0))
    DrawImage(ImageID(CibleImage), 0, 0)
  StopDrawing()
  
  Repeat
    Event = WaitWindowEvent()
  Until Event = #PB_Event_CloseWindow
EndIf

Option B : Utilisation Avancée (Via Structure de Contexte / Mode Ex)

Idéal pour manipuler les buffers mémoire, l'organisation de pointeurs, ou l'automatisation en chaîne.

IncludeFile "filtres.pbi"
UseModule filtres

If OpenWindow(0, 0, 0, 800, 600, "Exemple d'Appel Avancé Contextuel...", #PB_Window_SystemMenu | #PB_Window_ScreenCentered)
  
  SourceImage = 1
  CibleImage  = 2
   
  file$ = OpenFileRequester("Image","","",0)
  If load_image_32(SourceImage , file$) = 0
    End
  Else
    ResizeImage(SourceImage, 800, 600)
  EndIf
  
  CopyImage(SourceImage, CibleImage)
  
  ; 1. Nettoyage initial de la structure du contexte
  Clear_Data_Filter(FilterCtx)
  
  ; 2. Attribution des options spécifiques dans le tableau de configuration
  FilterCtx\option[0] = 5  ; Équivalent au Paramètre A
  FilterCtx\option[1] = 12 ; Équivalent au Paramètre B
  FilterCtx\option[2] = 0  ; Équivalent au ModeOption
  
  ; 3. Spécification de la source (3 syntaxes possibles selon vos besoins)
  Set_Source(SourceImage)
  ; ou Set_SourceEx(adresse_memoire_image , lg , ht)
  ; ou FilterCtx\image[0] = adresse_memoire_image (et spécifier image_lg[0] et image_ht[0])
  
  ; 4. Spécification de la cible
  Set_cible(CibleImage)
  ; ou Set_CibleEx(adresse_memoire_image , lg , ht) si gestion directe en mémoire
  
  ; Exécution du filtre étendu : Remplacer "NomDuFiltreEx" par le filtre désiré
  NomDuFiltreEx(FilterCtx)
  
  StartDrawing(WindowOutput(0))
    DrawImage(ImageID(CibleImage), 0, 0)
  StopDrawing()
  
  Repeat
    Event = WaitWindowEvent()
  Until Event = #PB_Event_CloseWindow
EndIf
