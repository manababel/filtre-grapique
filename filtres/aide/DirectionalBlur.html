Documentation - Algorithme DirectionalBoxBlur

Flou directionnel par accumulation fenêtrée et approximations de Gauss par passes successives

1. Pourquoi utiliser cette implémentation de Directional Box Blur ?

  • Approximation d'un flou directionnel gaussien : Selon le théorème central limite, répéter plusieurs fois un flou de boîte (Box Blur) donne un résultat très proche d'un flou gaussien. Le paramètre du nombre de passes (ndp) permet d'adoucir les traînées rectilignes pour obtenir un rendu soyeux à moindre coût mathématique.
  • Division optimisée par multiplication inverse : La division est l'une des opérations les plus gourmandes pour un processeur. Votre code calcule la division une seule fois par pixel (invCount = 1.0 / count) puis applique une simple multiplication (rSum * invCount) pour normaliser la moyenne, accélérant sensiblement le traitement.
  • Optimisation mémoire intelligente à l'écriture : La fonction maîtresse planifie la boucle de répétition du filtre de façon à réinjecter l'image de sortie comme source de la passe suivante (CopyMemory). Le code intègre un test de garde (If i < passes) pour s'épargner une copie mémoire totalement inutile lors du dernier cycle.

2. Les Inconvénients et Limites du Filtre

  • Atténuation du flou aux frontières de l'image : Contrairement au flou de bougé qui fige ses index sur les bords (Clamping), ce filtre choisit de rejeter les lectures hors-cadre (If sx >= 0 And sx < width...) et de recalculer la moyenne uniquement sur les pixels valides (variable count). Conséquence : le flou devient de moins en moins prononcé à mesure que l'on s'approche des bordures de la texture, créant une disparité visuelle.
  • Effet de bandes (Banding) persistant : Un flou de boîte distribue ses poids de manière uniforme. Si le nombre de passes est configuré à 1, les textures floutées laissent apparaître des lignes parallèles nettes et disgracieuses, typiques des filtres à moyenne brute.
  • Complexité cumulative des passes — O(N * (2R + 1) * P) : Pour obtenir un flou de qualité (3 passes), l'algorithme doit relire et retraiter l'intégralité des pixels trois fois de suite. Le coût machine initial lié au rayon se retrouve multiplié par le nombre de passes, ce qui nuit aux performances sur les gros rayons de traitement.

3. Structure et Intégration (PureBasic)

La Fonction d'Appel Publique

L'interface initialise les paramètres requis par le contexte moteur : l'angle en degrés, le rayon d'action et le nombre de passes d'adoucissement désirées.

Procedure DirectionalBoxBlur(source, cible, mask, angle, radius, ndp)
  Set_Source(source)
  Set_Cible(cible)
  Set_Mask(mask)
  With FilterCtx
    \option[0] = angle
    \option[1] = radius
    \option[2] = ndp
  EndWith
  DirectionalBoxBlurEx(FilterCtx.FilterParams)
EndProcedure

Gestion du Cycle de Répétition

Cette section supervise l'application des passes successives et gère le chaînage des buffers mémoire de façon optimale.

; Extrait de la boucle de traitement par passes :
Protected i, passes = \option[2]

For i = 1 To passes
  ; Exécute le traitement sur les axes obliques
  Create_MultiThread_MT(@DirectionalBoxBlur_MT())
  
  ; Si une autre passe suit, la destination devient la nouvelle source
  If i < passes
    CopyMemory(\addr[1], \addr[0], total)
  EndIf
Next

4. Le Filtre : Comment ça marche ?

Le filtre applique une fenêtre coulissante rectiligne symétrique (de -radius à +radius) orientée selon le vecteur trigonométrique calculé.

Le mécanisme du compteur dynamique (Count)

Parce que la ligne de flou exclut les pixels situés en dehors des coordonnées physiques de l'image, la taille de l'échantillon varie. Le programme ne divise pas par une constante fixe, mais comptabilise les points réels accumulés (count + 1) afin de garantir que la moyenne colorimétrique reste mathématiquement équilibrée.

Bilan d'échantillonnage asymétrique

Calcul du décalage des coordonnées :

Pas_X = Cos(Angle)  |  Pas_Y = Sin(Angle)

Pixel_Voisin(i) = (Pixel_X + i * Pas_X, Pixel_Y + i * Pas_Y)

La variable de boucle i évolue symétriquement autour du pixel central P. Le programme extrait la couleur de chaque voisin via un calcul d'adresse optimisé par décalage de bits gauche (<< 2), équivalent à une multiplication rapide par 4 octets.

Sécurité par rejet des coordonnées

Pour faire barrage aux violations d'accès mémoire sans ralentir la boucle par des sauts conditionnels multiples, la validation des coordonnées se fait sur une seule ligne :

If sx >= 0 And sx < width And sy >= 0 And sy < height

Les valeurs flottantes décimales (sx, sy) sont implicitement converties en valeurs entières lors de la sélection finale par la commande Int(), isolant uniquement les données contenues à l'intérieur de la matrice de l'image.

Logique Interne de l'Algorithme (Pseudo-code Synchrone)

Voici la chronologie séquentielle des opérations effectuées par le module de flou de boîte directionnel :

Entrée: Image Source, Image Cible, Angle, Rayon, Passes

// 1. PREPARATION DES PARAMETRES VECTORIELS
Angle_Radians = Angle * PI / 180.0
Pas_X = Cos(Angle_Radians)
Pas_Y = Sin(Angle_Radians)
Taille_Image_Octets = Largeur * Hauteur * 4

Allouer_Espace_Temporaire(*Buffer_Source, Taille_Image_Octets)
Copier_Mémoire(Image_Source -> *Buffer_Source)

// 2. ENCHAINEMENT DES PASSES
Pour cycle = 1 To Passes
    
    // Balayage de la surface de l'image
    Pour chaque Ligne Y de 0 à Hauteur - 1
        Pour chaque Colonne X de 0 à Largeur - 1
            
            Somme_R = 0 : Somme_G = 0 : Somme_B = 0 : Pixels_Valides = 0
            
            // Échantillonnage rectiligne symétrique
            Pour i = -Rayon To Rayon
                Voisin_X = X + i * Pas_X
                Voisin_Y = Y + i * Pas_Y
                
                // Rejet des points en dehors de l'image
                Si Voisin_X >= 0 Et Voisin_X < Largeur Et Voisin_Y >= 0 Et Voisin_Y < Hauteur
                    Pixel = Lire_Pixel(*Buffer_Source, Int(Voisin_X), Int(Voisin_Y))
                    Somme_R + Extraction_Rouge(Pixel)
                    Somme_G + Extraction_Vert(Pixel)
                    Somme_B + Extraction_Bleu(Pixel)
                    Pixels_Valides + 1
                Fin Si
            Next i
            
            // Calcul de la moyenne par inverse
            Si Pixels_Valides > 0
                Inverse_Multiplicateur = 1.0 / Pixels_Valides
                R_Final = Somme_R * Inverse_Multiplicateur
                G_Final = Somme_G * Inverse_Multiplicateur
                B_Final = Somme_B * Inverse_Multiplicateur
            Sinon
                R_Final = 0 : G_Final = 0 : B_Final = 0
            Fin Si
            
            Écrire_Pixel(Image_Cible, X, Y, Réassembler_ARGB(255, R_Final, G_Final, B_Final))
        Next X
    Next Y
    
    // Préparation de la passe suivante si nécessaire
    Si cycle < Passes
        Copier_Mémoire(Image_Cible -> *Buffer_Source)
    Fin Si

Next cycle

// 3. NETTOYAGE
Libérer_Mémoire(*Buffer_Source)
Fin
