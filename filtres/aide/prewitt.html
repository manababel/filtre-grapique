Documentation - Algorithme Prewitt

Analyse de la structure géométrique par détection des contours

1. Pourquoi utiliser l'opérateur de Prewitt ?

  • Détection de contours omnidirectionnelle (Boussole) : Contrairement aux filtres de gradient basiques qui n'analysent que les axes X et Y, l'implémentation Prewitt Boussole calcule le gradient maximum sur 8 directions distinctes (N, NE, E, SE, S, SO, O, NO). Cela garantit qu'aucune ligne ni transition, quelle que soit son orientation spatiale, ne soit ignorée.
  • Immunité relative au bruit local : En effectuant une sommation pondérée sur un voisinage de 3x3 pixels, l'opérateur de Prewitt intègre un lissage uniforme inhérent à sa structure. Cela lui permet de mieux tolérer le bruit haute fréquence qu'un simple filtre de dérivation de type Roberts.
  • Idéal pour la segmentation et le détourage : L'extraction des contours par Prewitt est une étape fondamentale pour l'analyse de formes, la vectorisation d'images, le traitement de cartographies ou la préparation d'un masque binaire d'accentuation (Unsharp Mask).

2. Exemples d'implémentation (PureBasic)

Option A : Utilisation Directe (Fonction Standard)

Idéal pour les appels simples et rapides avec passage de paramètres en arguments.

IncludeFile "filtres.pbi"
UseModule filtres

If OpenWindow(0, 0, 0, 800, 600, "Exemple Prewitt...", #PB_Window_SystemMenu | #PB_Window_ScreenCentered)
  
  SourceImage    = 1
  CibleImage     = 2
  AucunMasque    = 0
  Multiplicateur = 10 ; Ajuste la sensibilité/luminosité des contours (0 à 100)
  NoirEtBlanc    = 1  ; 1 pour convertir le gradient en niveaux de gris, 0 pour garder les contours colorés
  Inversion      = 0  ; 1 pour avoir les contours noirs sur fond blanc (pratique pour l'impression)
 
  file$ = OpenFileRequester("Image","","",0)
  If load_image_32(SourceImage , file$) = 0
    End
  Else
    ResizeImage(SourceImage, 800, 600)
  EndIf
  
  CopyImage(SourceImage, CibleImage)

  Prewitt(SourceImage, CibleImage, AucunMasque, Multiplicateur, NoirEtBlanc, Inversion)
  
  StartDrawing(WindowOutput(0))
  DrawImage(ImageID(CibleImage) ,  0 , 0)
  StopDrawing()
  
  Repeat
    Event = WaitWindowEvent()
    Select Event 
      Case #PB_Event_Gadget
        Select EventGadget()
          Case 1 
            CloseWindow(0)
            End  
        EndSelect
    EndSelect

  Until Event = #PB_Event_CloseWindow
EndIf

Option B : Utilisation Avancée (Via Structure de Contexte / PrewittEx)

Idéal pour la gestion de pointeurs mémoires complexes, les buffers personnalisés ou le traitement par lots en conservant un contexte de filtre.

IncludeFile "filtres.pbi"
UseModule filtres

If OpenWindow(0, 0, 0, 800, 600, "Exemple Prewitt Avancé...", #PB_Window_SystemMenu | #PB_Window_ScreenCentered)

  Else

  EndIf
  
  CopyImage(SourceImage, CibleImage)
  
  Clear_Data_Filter(FilterCtx)
  
  FilterCtx\option[0] = 10 ; Multiplicateur d'intensité du contour
  FilterCtx\option[1] = 1  ; Rendu en Noir et Blanc (Gris)
  FilterCtx\option[2] = 0  ; Inverser le résultat (0 = normal, 1 = inversé)
  
  Set_Source(SourceImage)
  Set_cible(CibleImage)
  
  PrewittEx(FilterCtx)
  
  StartDrawing(WindowOutput(0))
  DrawImage(ImageID(CibleImage) ,  0 , 0)
  StopDrawing()
  
  Repeat
    Event = WaitWindowEvent()
    Select Event 
      Case #PB_Event_Gadget
        Select EventGadget()
          Case 1 
            CloseWindow(0)
            End  
        EndSelect
    EndSelect

  Until Event = #PB_Event_CloseWindow
EndIf

3. Le Filtre : Comment ça marche ?

Le filtre de Prewitt est ce qu'on appelle un détecteur de contours géométriques. Son but fondamental est de scanner une image pixel par pixel pour repérer les endroits où les couleurs changent brusquement (par exemple, la frontière entre les cheveux sombres d'une personne et un mur blanc).

Pour faire cela de manière ultra-simple, le filtre utilise une méthode visuelle basée sur la comparaison de voisinage.

Le jeu des différences

Pour chaque point de la photo, le filtre regarde un petit carré de 3x3 pixels. Il va comparer la quantité de lumière d'un côté du carré par rapport au côté opposé :

  • Si les deux côtés ont la même luminosité (comme une zone de ciel bleu uni), la différence est de zéro. Le filtre comprend qu'il n'y a rien à voir et laisse la zone noire.
  • Si un côté est très clair et l'autre très sombre, la différence est très élevée. Le filtre détecte une cassure : il s'agit d'une ligne ou d'une bordure d'objet.

Le principe de la boussole directionnelle

L'originalité de cet algorithme est de chercher les lignes dans toutes les directions possibles (vers le Haut, le Bas, la Gauche, la Droite, et les 4 Diagonales), un peu comme les points cardinaux d'une boussole.

La règle mathématique visuelle :

Score_Du_Pixel = Écart_Maximum_Mesuré_(Côté_Clair_-_Côté_Sombre)

La ligne du milieu (0) sert de pivot. Les pixels du haut reçoivent un poids positif (+1) et ceux du bas un poids négatif (-1). Le filtre fait glisser ce calcul sur toute l'image.

Logique de rendu globale (Pseudo-code)

Voici le cheminement logique simplifié que suit l'algorithme sur l'ensemble de la surface graphique :

Pour chaque ligne de l'image
    Pour chaque pixel de cette ligne
        
        Écart_Le_Plus_Fort = 0
        
        // Analyser l'environnement sur 8 angles (boussole)
        Pour direction = 1 To 8
            Valeur_Calculée = Mesurer_Ecart_Lumière(pixel_actuel, direction)
            
            Si Valeur_Calculée > Écart_Le_Plus_Fort
                Écart_Le_Plus_Fort = Valeur_Calculée
            Fin Si
        Next
        
        // Allumer le pixel sur l'image finale
        Dessiner_Pixel_Contours(pixel_actuel, Écart_Le_Plus_Fort)
        
    Next
Fin Pour
