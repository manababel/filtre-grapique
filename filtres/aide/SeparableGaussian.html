Documentation - Algorithme SeparableGaussian

Flou gaussien complet à découplage d'axes et contrôle fin de l'écart-type (Sigma)

1. Pourquoi utiliser cette implémentation Gaussienne ?

  • Contrôle total du Sigma ($\sigma$) : Contrairement aux implémentations simplifiées où l'écart-type est implicitement lié au rayon, ce filtre permet de découpler mathématiquement la portée du flou (le rayon) et la forme de la cloche d'atténuation via la valeur `sigma_x10`. Si `sigma` est configuré à 0, le programme le calcule automatiquement par la formule standard : $$\sigma = \frac{\text{rayon}}{3}$$
  • Calcul du Rayon Effectif : La procédure optimise les cycles d'exécution en réévaluant le rayon utile (`CalcEffectiveRadius`). Si un grand rayon est demandé mais que la valeur de Sigma s'avère étroite, le filtre tronque la taille du noyau pour éliminer les multiplications inutiles par zéro en périphérie de la cloche.
  • Préservation complète de l'Alpha (ARGB) : L'algorithme décompone et traite le canal Alpha ($A$) de manière totalement isolée au même titre que les canaux colorimétriques ($R, G, B$), évitant ainsi les artefacts de transparence sur les contours des images détourées.

2. Les Inconvénients et Limites du Filtre

  • Surcharge des buffers flottants internes : Pour assurer une justesse arithmétique totale sans perte lors de l'accumulation, chaque ligne et chaque colonne de l'image est recopiée dans des tableaux de flottants dédiés (`lineR.f()`, `colR.f()`, etc.). Ces multiples va-et-vient entre la mémoire brute et les structures de variables locales ralentissent le pipeline par rapport à un accès direct par pointeur.
  • Complexité croissante avec le rayon : Le coût de traitement reste tributaire de la taille du noyau ($O(N \times R)$). Même si la séparation 1D limite les dégâts, l'augmentation du rayon induit inévitablement une baisse de réactivité de l'application, au contraire des algorithmes à temps constant comme le StackBlur.
  • Consommation mémoire doublée : L'obligation de figer la première passe (axe X) avant d'engager la seconde (axe Y) contraint le système à allouer un buffer temporaire (`*tmp`) équivalent à la taille totale de l'image en octets.

3. Structure et Intégration (PureBasic)

La Fonction d'Appel Publique

L'interface utilisateur transmet le rayon et le paramètre d'écart-type multiplié par 10 (pour éviter l'utilisation de valeurs flottantes dans les curseurs des interfaces graphiques).

Procedure SeparableGaussian(source, cible, mask, rayon, sigma_x10)
  Set_Source(source)
  Set_Cible(cible)
  Set_Mask(mask)
  With FilterCtx
    \option[0] = rayon
    \option[1] = sigma_x10
  EndWith
  ; Lance la validation et le calcul de sécurité
  SeparableGaussianEx(FilterCtx.FilterParams)
EndProcedure

Génération Dynamique du Noyau Normalisé

Cette routine calcule la distribution de probabilité 1D et s'assure que la somme des coefficients soit égale à $1.0$ afin de stabiliser l'énergie lumineuse globale.

Procedure CreateGaussianKernel_Sep(Array kernel.f(1), radius.l, sigma.f)
  If sigma <= 0.0 : sigma = radius / 3.0 : EndIf
  ; (...) Ajustement du rayon optimal
  Protected size = radius * 2 + 1
  Dim kernel(size - 1)
  
  Protected sigma2.f = 2.0 * sigma * sigma
  Protected sum.f = 0.0
  
  For i = 0 To size - 1
    x = i - radius
    kernel(i) = Exp(-(x * x) / sigma2) ; Équation simplifiée de Gauss 1D
    sum + kernel(i)
  Next
  
  If sum > 0.0
    For i = 0 To size - 1 : kernel(i) / sum : Next ; Normalisation
  IfEnd
EndProcedure

4. Le Filtre : Comment ça marche ?

Le traitement découple la matrice 2D en deux convolutions directionnelles successives.

Le secret de l'extraction de précision

Dans PureBasic, la manipulation directe des pixels en mémoire nécessite un masquage rigoureux bits à bits (`& $FF`) lors de l'extraction des composantes de couleur pour éviter l'interprétation de l'octet de poids fort comme un signe négatif. Une fois converties proprement en variables `Float`, les opérations mathématiques de mélange conservent toute leur précision avant la recomposition par décalage de bits (`<<`).

L'équation de convolution découplée :

$$G(x, y) = G(x) \times G(y) = \frac{1}{\sqrt{2\pi}\sigma} e^{-\frac{x^2}{2\sigma^2}} \times \frac{1}{\sqrt{2\pi}\sigma} e^{-\frac{y^2}{2\sigma^2}}$$

Le pixel P est d'abord échantillonné sur l'horizontale (X) depuis la source vers le buffer intermédiaire. La seconde passe applique le traitement sur la verticale (Y) depuis le buffer vers la destination finale.

Logique Interne de l'Algorithme (Pseudo-code Synchrone)

Voici l'enchaînement logique complet des deux passes du filtre :

Entrée: Image Source, Image Cible, Rayon, Paramètre Sigma

// 1. ÉTAPE DE SÉCURITÉ ET NOYAU
Sigma = Paramètre_Sigma / 10.0
Rayon_Effectif = Calculer_Rayon_Utile(Rayon, Sigma)
Générer_Et_Normaliser_Noyau_Gauss(Kernel_Tableau, Rayon_Effectif, Sigma)
Allouer_Buffer_Intermédiaire(*tmp, Largeur, Hauteur)

// 2. PASSE HORIZONTALE (Axe X) : Source ➔ Buffer (*tmp)
Pour chaque Ligne Y de 0 à Hauteur - 1
    // Étape A : Extraction et conversion en float de la ligne courante
    Pour chaque Colonne X de 0 à Largeur - 1
        Pixel = Lire_Pixel_Mémoire(Image_Source, X, Y)
        lineA(X) = Extraction_Alpha(Pixel)
        lineR(X) = Extraction_Rouge(Pixel)
        lineG(X) = Extraction_Vert(Pixel)
        lineB(X) = Extraction_Bleu(Pixel)
    Next X
    
    // Étape B : Convolution de la ligne
    Pour chaque Colonne X de 0 à Largeur - 1
        SumA = 0 : SumR = 0 : SumG = 0 : SumB = 0
        Pour dx = -Rayon_Effectif To Rayon_Effectif
            CibleX = X + dx
            Vérifier_Et_Ajuster_Bornes(CibleX, 0, Largeur - 1) // Clamping
            
            Coefficient = Kernel_Tableau[dx + Rayon_Effectif]
            SumA + lineA(CibleX) * Coefficient
            SumR + lineR(CibleX) * Coefficient
            SumG + lineG(CibleX) * Coefficient
            SumB + lineB(CibleX) * Coefficient
        Next dx
        
        Écrire_Pixel_Mémoire(*tmp, X, Y, Réassembler_ARGB(SumA, SumR, SumG, SumB))
    Next X
Fin Pour

// 3. PASSE VERTICALE (Axe Y) : Buffer (*tmp) ➔ Image Cible
Pour chaque Colonne X de 0 à Largeur - 1
    // Étape A : Extraction et conversion en float de la colonne courante depuis le buffer
    Pour chaque Ligne Y de 0 à Hauteur - 1
        Pixel = Lire_Pixel_Mémoire(*tmp, X, Y)
        colA(Y) = Extraction_Alpha(Pixel)
        colR(Y) = Extraction_Rouge(Pixel)
        colG(Y) = Extraction_Vert(Pixel)
        colB(Y) = Extraction_Bleu(Pixel)
    Next Y
    
    // Étape B : Convolution de la colonne
    Pour chaque Ligne Y de 0 à Hauteur - 1
        SumA = 0 : SumR = 0 : SumG = 0 : SumB = 0
        Pour dy = -Rayon_Effectif To Rayon_Effectif
            CibleY = Y + dy
            Vérifier_Et_Ajuster_Bornes(CibleY, 0, Hauteur - 1) // Clamping
            
            Coefficient = Kernel_Tableau[dy + Rayon_Effectif]
            SumA + colA(CibleY) * Coefficient
            SumR + colR(CibleY) * Coefficient
            SumG + colG(CibleY) * Coefficient
            SumB + colB(CibleY) * Coefficient
        Next dy
        
        Écrire_Pixel_Mémoire(Image_Cible, X, Y, Réassembler_ARGB(SumA, SumR, SumG, SumB))
    Next Y
Fin Pour

// 4. LIBÉRATION
Libérer_Buffer(*tmp)
Fin
