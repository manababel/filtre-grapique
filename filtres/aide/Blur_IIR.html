Documentation - Algorithme Blur IIR

Flou récursif à Réponse Impulsionnelle Infinie à vitesse constante

1. Pourquoi utiliser le flou IIR ?

  • Performance O(1) de type Gaussien : Tout comme la technique du Box Blur SAT, le temps d'exécution d'un filtre IIR est totalement indépendant de la taille du rayon d'action (rx, ry). Cependant, là où le Box Blur produit un flou angulaire "au rendu carré", l'IIR génère nativement une approximation fluide et mathématiquement superbe du Flou Gaussien.
  • Économie de mémoire Drastique : Contrairement à la Summed-Area Table qui nécessite l'allocation de 4 immenses buffers de données en 64 bits, le filtre IIR traite l'image de manière séquentielle en n'utilisant que quelques variables tampons pour propager le calcul.
  • Nombre de Passes Ajustable (ndp) : Le paramètre de passe permet de choisir le compromis parfait entre vitesse et qualité. Une seule passe offre un flou ultra-rapide, tandis que 2 ou 3 passes affinent la courbe pour obtenir une approximation parfaite de l'authentique cloche de Gauss.

2. Exemples d'implémentation (PureBasic)

Option A : Utilisation Directe (Fonction Standard)

Idéal pour une intégration simple, cette procédure configure automatiquement l'ensemble des paramètres requis par le moteur de rendu.

IncludeFile "filtres.pbi"
UseModule filtres

If OpenWindow(0, 0, 0, 800, 600, "Exemple IIR...", #PB_Window_SystemMenu | #PB_Window_ScreenCentered)
  
  SourceImage = 1
  CibleImage  = 2
  AucunMasque = 0
  RayonX      = 15  ; Intensité du flou horizontal
  RayonY      = 15  ; Intensité du flou vertical
  Passes      = 1   ; 'ndp' : Nombre de passes récursives (souvent égal à 1, 2 ou 3)
 
  file$ = OpenFileRequester("Ouvrir une Image","","",0)
  If load_image_32(SourceImage , file$) = 0
    End
  Else
    ResizeImage(SourceImage, 800, 600)
  EndIf
  
  CopyImage(SourceImage, CibleImage)

  ; Appel de la procédure avec ses paramètres explicites
  Blur_IIR(SourceImage, CibleImage, AucunMasque, RayonX, RayonY, Passes)
  
  StartDrawing(WindowOutput(0))
    DrawImage(ImageID(CibleImage) ,  0 , 0)
  StopDrawing()
  
  Repeat
    Event = WaitWindowEvent()
  Until Event = #PB_Event_CloseWindow
EndIf

3. Le Filtre : Comment ça marche ?

Un filtre IIR (Infinite Impulse Response) utilise la réactivité des pixels précédents pour calculer le pixel actuel. C'est le principe du **filtre à rétroaction**. La couleur d'un pixel est "injectée" et se dissout progressivement dans ses voisins au fil du balayage.

Le concept de la propagation d'onde

Imaginez que vous fassiez glisser un pinceau mouillé de gauche à droite sur une suite de tâches de couleur. Le pinceau accumule la couleur de la tâche précédente et la dépose doucement sur la tâche suivante. Pour éviter que le flou ne bave que vers la droite, l'algorithme fait immédiatement la même chose en sens inverse (de droite à gauche).

L'importance des Balayages Bidirectionnels

Pour obtenir un flou parfaitement symétrique et homogène sans décalage visuel, l'algorithme Blur_IIREx effectue des passes croisées sur les lignes et les colonnes :

L'équation de récursion d'ordre 2 classique :

Pixel_Flou = (a0 * Pixel_Brut) + (b1 * Pixel_Precedent_1) + (b2 * Pixel_Precedent_2)

La couleur finale du pixel dépend d'un ratio de sa propre couleur originale, mélangé avec l'historique des deux pixels qui viennent d'être traités juste avant lui. Les coefficients mathématiques ($a, b$) sont calculés au démarrage selon le rayon ($rx, ry$).

Logique interne de l'algorithme (Pseudo-code)

Voici la structure logique générale appliquée lors de l'exécution de Blur_IIREx :

// Calcul initial des coefficients mathématiques basés sur le Rayon
Calculer_Coefficients_Filtre(RayonX, RayonY)

Pour passe = 1 To ndp  // Répétition selon le nombre de passes demandé

    // --- PASSE HORIZONTALE ---
    Pour chaque ligne Y de l'image
        // Passe Causal (Aller : de gauche à droite)
        Pour X = 0 To Largeur - 1
            Buffer_Aller[X] = (Coeff_A * Pixel_Source) + (Coeff_B1 * Buffer_Aller[X-1]) + ...
        Next
        
        // Passe Anti-Causal (Retour : de droite à gauche)
        Pour X = Largeur - 1 To 0 Step -1
            Buffer_Retour[X] = (Coeff_A * Buffer_Aller[X]) + (Coeff_B1 * Buffer_Retour[X+1]) + ...
        Next
    Fin Pour

    // --- PASSE VERTICALE ---
    Pour chaque colonne X de l'image
        // Passe Causal (Aller : du haut vers le bas)
        Pour Y = 0 To Hauteur - 1
            Buffer_Vertical_Aller[Y] = (Coeff_A * Pixel_Intermediaire) + (Coeff_B1 * Buffer_Vertical_Aller[Y-1]) + ...
        Next
        
        // Passe Anti-Causal (Retour : du bas vers le haut)
        Pour Y = Hauteur - 1 To 0 Step -1
            Pixel_Final[X, Y] = (Coeff_A * Buffer_Vertical_Aller[Y]) + (Coeff_B1 * Buffer_Vertical_Retour[Y+1]) + ...
        Next
    Fin Pour
