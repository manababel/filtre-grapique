Échantillonnage Centripète

Documentation - Algorithme RadialBlur

Effet tunnel dynamique par intégration linéaire directionnelle vers un point d'impact

1. Pourquoi utiliser cette implémentation de Flou Radial ?

  • Coordonnées Relatives en Pourcentage : L'algorithme convertit automatiquement les valeurs curseurs (0 à 100) en coordonnées réelles de pixels (cx, cy) par rapport à la taille de la texture. Cela garantit le maintien du point focal (le centre du flou) même si l'image source change d'échelle.
  • Passe de Garde Circulaire (Early Out) : Pour maximiser la vitesse d'exécution, le filtre évalue en amont la distance au carré de chaque pixel par rapport au point d'impact. Si le pixel se trouve en dehors du rayon d'action maximal (dist > rmax2), la boucle de flou est instantanément court-circuitée (Continue) au profit d'une copie directe de la mémoire brute.
  • Atténuation Linéaire Intégrée (Force) : Le mélange entre le pixel d'origine et le pixel flouté est piloté par un facteur d'atténuation. Plus le pixel est proche du centre (cx, cy), plus le flou est maximal. Plus il s'en éloigne, plus il se fond doucement avec la netteté d'origine avant de s'annuler au-delà de rmax.

2. Les Inconvénients et Limites du Filtre

  • Effet de pixellisation (Aliasing) crénelé en périphérie : Comme le vecteur de déplacement d'échantillonnage (dxStep, dyStep) est calculé en fonction de la distance séparant le pixel courant du centre, l'écart entre chaque échantillon s'accroît considérablement à mesure qu'on s'éloigne du foyer. Avec un nombre d'échantillons faible, le filtre affiche des motifs géométriques hachés et espacés au lieu d'un flou lisse.
  • Coût machine lié à la qualité (Rayon = Échantillons) : Le paramètre de rayon (Radius) ne définit pas une distance spatiale mais le nombre d'itérations de la boucle interne de sous-échantillonnage. Pousser la qualité à 50 signifie que pour chaque pixel de l'image (non éliminé par la garde), le filtre doit lire et accumuler 50 pointeurs mémoire.
  • Perte d'énergie aux limites par rejet de bords : Lorsque le vecteur de balayage se rapproche des bords de l'image, certains sous-échantillons se retrouvent hors des limites de la matrice. Le filtre rejette ces lectures en toute sécurité (If sx >= 0...) mais sans ajuster le dénominateur de la moyenne finale (qui reste fixé sur la constante samp), ce qui peut provoquer une baisse anormale de luminosité (un assombrissement) sur les pourtours.

3. Structure et Intégration (PureBasic)

La Fonction d'Appel Publique

L'interface récupère le nombre d'échantillons, les coordonnées du centre du flou (en %), ainsi que la taille de la zone d'influence.

Procedure RadialBlur(source, cible, mask, echantillonnage, posx, posy, rmax)
  Set_Source(source)
  Set_Cible(cible)
  Set_Mask(mask)
  With FilterCtx
    \option[0] = echantillonnage
    \option[1] = posx
    \option[2] = posy
    \option[3] = rmax
  EndWith
  RadialBlurEx(FilterCtx.FilterParams)
EndProcedure

Initialisation géométrique et facteurs d'échelle

Cette phase pré-calcule le repère focal de l'image et détermine la barrière de rejet circulaire à l'aide de l'hypoténuse de l'image.

; Calcul du centre de gravité et du rayon d'action au carré :
Protected cx = (\option[1] * lg) / 100
Protected cy = (\option[2] * ht) / 100
Protected rmax = (\option[3] * Sqr(lg*lg + ht*ht)) / 100
Protected rmax2.f = rmax * rmax

; Facteur multiplicateur inverse d'accumulation (évite la division finale)
Protected samp.f = 1.0 / (Radius + 1)

4. Le Filtre : Comment ça marche ?

Pour chaque pixel de l'image, l'algorithme calcule un vecteur pointant directement vers le centre personnalisé, puis marche le long de ce chemin.

Le pas de déplacement dynamique (dxStep, dyStep)

La trajectoire d'échantillonnage est propre à chaque coordonnée. Le pas est défini par la distance résiduelle divisée par le nombre d'échantillons attendus :
dxStep = (cx - x) * samp et dyStep = (cy - y) * samp.
Le curseur de lecture démarre à la position d'origine du pixel (fx = x) et migre progressivement vers le centre à chaque itération de la sous-boucle.

Cinématique de l'aspiration radiale

Centre

L'équation du mélange pondéré (Mix) :

Force = (RayonMax² - Distance²) / RayonMax²

Couleur_Finale = Flou * Force + Original * (1.0 - Force)

Si un point se situe très près du Centre, sa distance tend vers 0. La variable force s'approche de 1.0, appliquant le flou de traînée au maximum. À la frontière du cercle d'influence, force s'annule, restaurant l'image nette originale.

Logique Interne de l'Algorithme (Pseudo-code Synchrone)

Voici la chronologie séquentielle du moteur d'intégration centripète :

Entrée: Image Source, Image Cible, Échantillonnage (Radius), Centre X (%), Centre Y (%), Rayon Max (%)

// 1. PRE-CALCULS GEOMETRIQUES
Foyer_X = (Centre_X * Largeur_Image) / 100
Foyer_Y = (Centre_Y * Hauteur_Image) / 100
Diagonale = Sqr(Largeur_Image^2 + Hauteur_Image^2)
Limite_Rayon = (Rayon_Max * Diagonale) / 100
Limite_Rayon_Carre = Limite_Rayon * Limite_Rayon

Inverse_Nombre_Echantillons = 1.0 / (Radius + 1)

// 2. TRAITEMENT DE LA SURFACE
Pour chaque Ligne Y de 0 à Hauteur - 1
    Delta_Y = Y - Foyer_Y
    
    Pour chaque Colonne X de 0 à Largeur - 1
        Delta_X = X - Foyer_X
        Distance_Carre = (Delta_X * Delta_X) + (Delta_Y * Delta_Y)
        
        // Étape A : Test de garde précoce
        Si Distance_Carre > Limite_Rayon_Carre
            Pixel_Net = Lire_Pixel(Image_Source, X, Y)
            Écrire_Pixel(Image_Cible, X, Y, Pixel_Net)
            Continuer au pixel suivant
        Fin Si
        
        // Étape B : Calcul du vecteur de déplacement vers le foyer
        Pas_X = (Foyer_X - X) * Inverse_Nombre_Echantillons
        Pas_Y = (Foyer_Y - Y) * Inverse_Nombre_Echantillons
        
        Curseur_X = X
        Curseur_Y = Y
        Somme_R = 0 : Somme_G = 0 : Somme_B = 0
        
        // Étape C : Échantillonnage le long du rayon
        Pour i = 0 To Radius
            Int_X = Int(Curseur_X)
            Int_Y = Int(Curseur_Y)
            
            Si Int_X >= 0 Et Int_X < Largeur Et Int_Y >= 0 Et Int_Y < Hauteur
                Pixel_Echantillon = Lire_Pixel(Image_Source, Int_X, Int_Y)
                Somme_R + Extraction_Rouge(Pixel_Echantillon)
                Somme_G + Extraction_Vert(Pixel_Echantillon)
                Somme_B + Extraction_Bleu(Pixel_Echantillon)
            Fin Si
            
            Curseur_X + Pas_X
            Curseur_Y + Pas_Y
        Next i
        
        // Étape D : Normalisation et interpolation de force
        Flou_R = Somme_R * Inverse_Nombre_Echantillons
        Flou_G = Somme_G * Inverse_Nombre_Echantillons
        Flou_B = Somme_B * Inverse_Nombre_Echantillons
        
        Facteur_Force = (Limite_Rayon_Carre - Distance_Carre) / Limite_Rayon_Carre
        Si Facteur_Force < 0 : Facteur_Force = 0 : Fin Si
        
        (Alpha, Orig_R, Orig_G, Orig_B) = Extraire_ARGB(Lire_Pixel(Image_Source, X, Y))
        
        R_Final = Flou_R * Facteur_Force + Orig_R * (1.0 - Facteur_Force)
        G_Final = Flou_G * Facteur_Force + Orig_G * (1.0 - Facteur_Force)
        B_Final = Flou_B * Facteur_Force + Orig_B * (1.0 - Facteur_Force)
        
        Écrire_Pixel(Image_Cible, X, Y, Réassembler_ARGB(Alpha, Clamping(R_Final), Clamping(G_Final), Clamping(B_Final)))
        
    Next X
Fin Pour
