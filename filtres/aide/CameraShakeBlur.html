Convolution par Vecteurs Empiriques

Documentation - Algorithme CameraShakeBlur

Simulation cinématique d'obturateur par échantillonnage de chemins stochastiques et harmoniques

1. Pourquoi utiliser cette implémentation de Flou de Bougé ?

  • Modélisation Cinématique Réaliste : Contrairement aux flous de mouvement classiques qui opèrent selon une ligne droite immuable, cet algorithme simule les micro-déplacements physiques des mains d'un photographe en contraignant le noyau de flou le long d'une structure de chemin personnalisée (ShakePath).
  • Trois Générateurs de Profils intégrés :
    • Aléatoire (Mouvement brownien) : Simule le tremblement organique à l'aide d'un vecteur cumulatif bridé par l'intensité maximale via une correction de rayon (px * intensity / radius).
    • Sinusoïdal (Oscillation) : Recrée les vibrations mécaniques cycliques en combinant des fréquences asynchrones (Cosinus à fréquence triple sur X, Sinus à fréquence double sur Y).
    • Circulaire avec Variation : Calcule une orbite cycloïdale complexe dont le rayon change continuellement en fonction du temps à l'aide d'une harmonique secondaire (0.5 * Sin(t * 5)).
  • Décorrélation Temporelle Déterministe : L'intégration d'un contrôle de graine aléatoire (RandomSeed) permet de figer le comportement du mode stochastique. L'utilisateur peut ainsi reproduire à l'identique le même flou de bougé d'un rendu à l'autre ou laisser le hasard gérer l'effet en injectant l'horloge système (ElapsedMilliseconds()).
  • Noyau Disjoint Non-Régulier : Le traitement lit directement les coordonnées décalées stockées dans le tableau de déplacement. Le filtre peut ainsi lisser l'image selon des motifs non contigus, reproduisant fidèlement les sauts de position d'un capteur optique en mouvement.

2. Les Inconvénients et Limites du Filtre

  • Défaut Critique d'Atténuation (Image Noire) : Dans la procédure de liaison CameraShakeBlurEx, la copie du chemin de déplacement vers la zone mémoire partagée est soumise à la condition If \option[3] <> 0 (l'option d'atténuation). Si l'utilisateur laisse le paramètre "Atténuation" à sa valeur par défaut de 0, la mémoire tampon reste vierge. Le filtre additionne alors des décalages nuls, et si la condition count > 0 échoue, le rendu final est altéré.
  • Saut de Ligne Inférieur : À l'instar de l'algorithme de torsion, la borne supérieure de la boucle principale d'analyse s'arrête à thread_stop - 1, ce qui risque d'exclure du calcul la dernière rangée de pixels de la surface de l'image.

3. Structure et Intégration (PureBasic)

La Structure Vectorielle de Déplacement

Les écarts géométriques (X et Y) appliqués à chaque échantillon de l'obturateur sont stockés dans une structure simple à mémoire contiguë :

Structure ShakePath
  x.l
  y.l
EndStructure

La Fonction d'Appel Publique

L'interface distribue les paramètres utilisateurs pour définir la trajectoire :

Procedure CameraShakeBlur(source, cible, mask, samples, intensite, pattern, attenuation, seed)
  Set_Source(source)
  Set_Cible(cible)
  Set_Mask(mask)
  With FilterCtx
    \option[0] = samples
    \option[1] = intensite
    \option[2] = pattern
    \option[3] = attenuation
    \option[4] = seed
  EndWith
  CameraShakeBlurEx(FilterCtx.FilterParams)
EndProcedure

4. Le Filtre : Comment ça marche ?

Avant d'analyser l'image, l'algorithme génère une liste fermée d'offsets de coordonnées. Pour chaque pixel de l'image, il applique simultanément l'ensemble de ces décalages pour accumuler les couleurs associées et en faire la moyenne.

Principe de Captation Stochastique

Pendant la phase de convolution, le pixel courant (i, j) n'interroge pas ses voisins immédiats sous forme de matrice carrée. Il va chercher le pixel déplacé à la coordonnée exacte (i + path(k)\x, j + path(k)\y).
Le motif global du flou dépend exclusivement du cas choisi lors de la phase de calcul initialisée par le générateur de chemin.

Cinématique du Flou de Bougé

Tracé du Path
k=0
i,jk=1
k=2

Formule d'Échantillonnage Disjoint :

Position_X_Echantillon = i + path(k)\x
Position_Y_Echantillon = j + path(k)\y

Si la coordonnée projetée se situe dans le cadre de l'image, le bit ARGB est extrait par décalage de bits. Si le compteur de points valides reste nul à la fin de la boucle k, le pixel d'origine est automatiquement préservé.

Logique Interne de l'Algorithme (Pseudo-code Synchrone)

Voici la logique d'exécution complète pour générer le flou d'obturateur :

Entrée: Image Source, Image Cible, Nb_Echantillons, Intensité, Mode_Pattern, Atténuation, Graine_Seed

// 1. INITIALISATION DU GENERATEUR ALEATOIRE ET DES TABLEAUX
Si Graine_Seed > 0 Alors
    Fixer_Graine_Alea(Graine_Seed)
Sinon
    Fixer_Graine_Alea(Temps_Systeme_Millisecondes)
Fin Si

Allouer_Tableau_Structure(Chemin_Shake, Type: ShakePath, Taille: Nb_Echantillons)

// 2. GENERATION DU MOTIF DE DEPLACEMENT (DEBUT DU GENERATEUR)
Sélectionner Mode_Pattern
    
    Cas 0: // TRAJECTOIRE ALEATOIRE (BROWN_MOTION)
        Pos_X_Alea = 0.0 : Pos_Y_Alea = 0.0
        Pour i = 0 À Nb_Echantillons - 1
            Pos_X_Alea = Pos_X_Alea + (Generer_Alea(-100, 100) / 100.0) * Intensité * 0.3
            Pos_Y_Alea = Pos_Y_Alea + (Generer_Alea(-100, 100) / 100.0) * Intensité * 0.3
            Rayon_Courant = Sqr(Pos_X_Alea^2 + Pos_Y_Alea^2)
            
            // Confinement dans la sphère d'intensité
            Si Rayon_Courant > Intensité Alors
                Pos_X_Alea = Pos_X_Alea * Intensité / Rayon_Courant
                Pos_Y_Alea = Pos_Y_Alea * Intensité / Rayon_Courant
            Fin Si
            Chemin_Shake[i]\x = Entier(Pos_X_Alea)
            Chemin_Shake[i]\y = Entier(Pos_Y_Alea)
        Next
        
    Cas 1: // TRAJECTOIRE SINUSOIDALE (OSCILLATION HARMONIQUE)
        Pour i = 0 À Nb_Echantillons - 1
            Temps_Normalisé = (i / (Nb_Echantillons - 1.0)) * 2 * 3.14159265
            Chemin_Shake[i]\x = Entier(Cos(Temps_Normalisé * 3) * Intensité)
            Chemin_Shake[i]\y = Entier(Sin(Temps_Normalisé * 2) * Intensité * 0.7)
        Next
        
    Cas 2: // TRAJECTOIRE CIRCULAIRE MODULEE
        Pour i = 0 À Nb_Echantillons - 1
            Temps_Normalisé = (i / (Nb_Echantillons - 1.0)) * 2 * 3.14159265
            Rayon_Modulaire = Intensité * (0.5 + 0.5 * Sin(Temps_Normalisé * 5))
            Chemin_Shake[i]\x = Entier(Cos(Temps_Normalisé) * Rayon_Modulaire)
            Chemin_Shake[i]\y = Entier(Sin(Temps_Normalisé) * Rayon_Modulaire)
        Next
        
Fin Sélection

// 3. CONVOLUTION ET APPLICATION DU FILTRE SUR L'IMAGE
Pour chaque Ligne Y de 0 à Hauteur_Image - 1
    Pour chaque Colonne X de 0 à Largeur_Image - 1
        
        Somme_A = 0 : Somme_R = 0 : Somme_G = 0 : Somme_B = 0
        Compteur_Points = 0
        
        Pour k = 0 À Nb_Echantillons - 1
            
            Echantillon_X = X + Chemin_Shake[k]\x
            Echantillon_Y = Y + Chemin_Shake[k]\y
            
            // Vérification des limites physiques de la texture
            Si Echantillon_X >= 0 Et Echantillon_X < Largeur_Image Et Echantillon_Y >= 0 Et Echantillon_Y < Hauteur_Image Alors
                
                Pixel_Couleur = Lire_Pixel_Source_Direct(Echantillon_X, Echantillon_Y)
                
                Somme_A = Somme_A + Extraire_Alpha(Pixel_Couleur)
                Somme_R = Somme_R + Extraire_Rouge(Pixel_Couleur)
                Somme_G = Somme_G + Extraire_Vert(Pixel_Couleur)
                Somme_B = Somme_B + Extraire_Bleu(Pixel_Couleur)
                Compteur_Points = Compteur_Points + 1
                
            Fin Si
        Next k
        
        // Calcul de la moyenne ou récupération du pixel d'origine (Fallback)
        Si Compteur_Points > 0 Alors
            A_Final = Somme_A / Compteur_Points
            R_Final = Somme_R / Compteur_Points
            G_Final = Somme_G / Compteur_Points
            B_Final = Somme_B / Compteur_Points
        Sinon
            Pixel_Origine = Lire_Pixel_Source_Direct(X, Y)
            A_Final = Extraire_Alpha(Pixel_Origine)
            R_Final = Extraire_Rouge(Pixel_Origine)
            G_Final = Extraire_Vert(Pixel_Origine)
            B_Final = Extraire_Bleu(Pixel_Origine)
        Fin Si
        
        Écrire_Pixel_Cible(X, Y, Réassembler_ARGB(A_Final, R_Final, G_Final, B_Final))
        
    Next X
Fin Pour

Libérer_Tableau(Chemin_Shake)
