Documentation - Algorithme CircularMeanblur

Flou artistique de moyenne circulaire par délimitation de masque en disque

1. Pourquoi utiliser le CircularMeanblur ?

  • Simulation de Bokeh Optique : Contrairement au Box Blur dont le balayage rectangulaire étire les détails le long des axes X et Y, le flou circulaire mélange les pixels de manière parfaitement isotrope (uniforme dans toutes les directions). Il imite la forme d'un diaphragme d'appareil photo circulaire ouvert, créant des flous d'arrière-plan très esthétiques.
  • Préservation des formes organiques : En éliminant les angles droits lors de la phase de calcul, ce filtre adoucit les contours sans générer l'effet de "grille pixelisée" ou de "bandes fantômes" propre aux convolutions carrées standards.
  • Calcul de distance Euclidienne : L'algorithme isole précisément les pixels environnants à l'aide de l'équation du cercle. Seuls les points dont la distance par rapport au centre est inférieure ou égale au rayon sont invités dans le calcul de la moyenne.

2. Exemples d'implémentation (PureBasic)

Option A : Utilisation Directe (Fonction Standard)

Idéal pour exécuter un flou circulaire simple et uniforme directement sur l'image cible via un rayon global.

IncludeFile "filtres.pbi"
UseModule filtres

If OpenWindow(0, 0, 0, 800, 600, "Exemple Flou Circulaire...", #PB_Window_SystemMenu | #PB_Window_ScreenCentered)
  
  SourceImage = 1
  CibleImage  = 2
  AucunMasque = 0
  Rayon       = 8   ; Rayon du disque de lissage
 
  file$ = OpenFileRequester("Sélectionner l'image","","",0)
  If load_image_32(SourceImage , file$) = 0
    End
  Else
    ResizeImage(SourceImage, 800, 600)
  EndIf
  
  CopyImage(SourceImage, CibleImage)

  ; Lancement du traitement circulaire
  CircularMeanblur(SourceImage, CibleImage, AucunMasque, Rayon)
  
  StartDrawing(WindowOutput(0))
    DrawImage(ImageID(CibleImage) ,  0 , 0)
  StopDrawing()
  
  Repeat
    Event = WaitWindowEvent()
  Until Event = #PB_Event_CloseWindow
EndIf

Option B : Traitement par l'exécuteur étendu

L'argument est injecté au sein de la structure de contexte avant le déploiement de la fonction noyau :

With FilterCtx

3. Le Filtre : Comment ça marche ?

Le principe fondamental reste le calcul d'une moyenne locale, mais la boîte de voisinage géométrique devient un **disque**.

Le test d'inclusion du Pixel

Pour chaque pixel situé dans la zone d'étude, l'algorithme calcule sa distance par rapport au pixel central (X, Y). Si le résultat respecte le théorème de Pythagore réadapté au cercle, la couleur est enregistrée pour la moyenne, sinon elle est purement ignorée.

Comparaison du voisinage (Masque de Rayon = 2)

La grille ci-dessous montre comment l'algorithme exclut les quatre coins de la boîte pour former le cercle :

11111

La condition mathématique d'inclusion :

(dx * dx) + (dy * dy) <= Rayon * Rayon

Les cases marquées 0 (les coins) échouent au test car elles sont géométriquement trop éloignées du centre (C). Seules les cases bleues (1) participent au mélange. Le diviseur final correspond exactement au nombre de cases validées.

Logique interne de l'algorithme (Pseudo-code)

Voici le parcours itératif effectué par la procédure CircularMeanblurEx :

Pour chaque ligne Y de la photo
    Pour chaque pixel X de cette ligne
        
        SommeA = 0 : SommeR = 0 : SommeG = 0 : SommeB = 0
        CompteurValid = 0
        
        // Parcourir le bloc englobant (Carré temporaire de côté = Rayon * 2)
        Pour dy = -rayon To rayon
            Pour dx = -rayon To rayon
                
                // Validation mathématique de la forme du disque
                Si (dx * dx) + (dy * dy) <= rayon * rayon
                    
                    VoisinX = X + dx
                    VoisinY = Y + dy
                    
                    // Vérification de sécurité (ne pas déborder des bords de l'image)
                    Si VoisinX dans les limites Et VoisinY dans les limites
                        SommeA + Pixel_Voisin_Alpha
                        SommeR + Pixel_Voisin_Rouge
                        SommeG + Pixel_Voisin_Vert
                        SommeB + Pixel_Voisin_Bleu
                        CompteurValid + 1
                    Fin Si
                    
                Fin Si
                
            Next dx
        Next dy
        
        // Calcul final de la moyenne locale du disque
        MoyenneA = SommeA / CompteurValid
        MoyenneR = SommeR / CompteurValid
        MoyenneG = SommeG / CompteurValid
        MoyenneB = SommeB / CompteurValid
        
        // Écriture finale sur l'image cible
        Pixel_Cible[X, Y] = ARGB(MoyenneA, MoyenneR, MoyenneG, MoyenneB)
        
    Next X
Fin Pour
