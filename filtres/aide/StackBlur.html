Algorithme Hybride Optimisé

Documentation - Algorithme StackBlur

Le flou haute qualité ultra-rapide par pile glissante pondérée

1. Pourquoi utiliser le StackBlur ?

  • Le meilleur des deux mondes (Vitesse et Qualité) : Le Box Blur standard est rapide mais crée des artefacts carrés disgracieux. Le Flou Gaussien est somptueux mais extrêmement lourd sur les grands rayons. Le StackBlur résout ce dilemme : il génère un rendu organique, doux et quasi-indissociable du flou gaussien, tout en s'exécutant en un temps constant $O(1)$.
  • Une pile glissante ultra-intelligente : Au lieu de faire des multiplications de matrices complexes à chaque pixel, le StackBlur maintient un registre (une pile) de pixels. Lorsqu'il se déplace d'un pixel vers la droite, il supprime un pixel du bout de la pile, ajoute le nouveau pixel entrant et met à jour la moyenne par de simples additions/soustractions.
  • Pondération linéaire naturelle : La magie de l'algorithme réside dans le fait que les pixels les plus proches du centre de la boîte ont un poids plus important dans la moyenne que les pixels situés sur les bords extérieurs, imitant ainsi naturellement la cloche du flou gaussien.

2. Exemples d'implémentation (PureBasic)

Option A : Utilisation Directe (Fonction Standard)

Idéal pour une intégration immédiate avec un contrôle indépendant du flou sur les axes X et Y.

IncludeFile "filtres.pbi"
UseModule filtres

If OpenWindow(0, 0, 0, 800, 600, "Exemple StackBlur...", #PB_Window_SystemMenu | #PB_Window_ScreenCentered)
  
  SourceImage = 1
  CibleImage  = 2
  AucunMasque = 0
  RayonX      = 20  ; Rayon du flou horizontal
  RayonY      = 20  ; Rayon du flou vertical
  Passes      = 1   ; 'ndp' : Nombre d'itérations (généralement fixé à 1)
 
  file$ = OpenFileRequester("Sélectionner une Image","","",0)
  If load_image_32(SourceImage , file$) = 0
    End
  Else
    ResizeImage(SourceImage, 800, 600)
  EndIf
  
  CopyImage(SourceImage, CibleImage)

  ; Exécution du filtre StackBlur standard
  StackBlur(SourceImage, CibleImage, AucunMasque, RayonX, RayonY, Passes)
  
  StartDrawing(WindowOutput(0))
    DrawImage(ImageID(CibleImage) ,  0 , 0)
  StopDrawing()
  
  Repeat
    Event = WaitWindowEvent()
  Until Event = #PB_Event_CloseWindow
EndIf

Option B : Passage des arguments via Contexte

En coulisses, la procédure configure l'environnement de la manière suivante avant d'appeler l'exécuteur optimisé :

With FilterCtx
  \option[0] = rx  // Affectation du rayon X
  \option[1] = ry  // Affectation du rayon Y
  \option[2] = ndp // Nombre de passes (ndp)
EndWith
StackBlurEx(FilterCtx.FilterParams)

3. Le Filtre : Comment ça marche ?

L'algorithme du StackBlur crée une pile de pixels sous forme de file d'attente. La taille de cette pile est égale à (Rayon * 2) + 1.

Le secret : La double accumulation

Pour éviter toute multiplication lors du calcul de la pondération, l'algorithme utilise trois variables de stockage glissantes : la Somme des Pixels de la pile, la Somme de la moitié supérieure de la pile (les pixels entrants) et la Somme de la moitié inférieure (les pixels sortants). Le déplacement d'un pixel met à jour ces trois variables en cascade par de simples opérations arithmétiques basiques.

Visualisation du mécanisme de la Pile Glissante

Pendant que la boîte se déplace sur une ligne, la pile fait glisser son contenu comme un tapis roulant :

Pondération triangulaire appliquée :

Intensité = (Pixel_Milieu * 2) + Pixel_Sort + Pixel_Entre

Le pixel central (indigo) est compté deux fois (poids maximal). Les pixels périphériques décroissent en importance. En divisant le total par la somme des poids (ici 4), on obtient un lissage parfait, sans aucun effet d'escalier sur les dégradés.

Logique interne de l'algorithme (Pseudo-code)

Voici la structure logique simplifiée exécutée de manière séquentielle dans le moteur StackBlurEx :

Pour passe = 1 To ndp

    // --- PASSE 1 : BALAYAGE HORIZONTAL (LIGNE PAR LIGNE) ---
    Pour chaque ligne Y de l'image
        Initialiser_Pile_Avec_Pixel_De_Gauche()
        Calculer_Sommes_Initiales_De_La_Pile()
        
        Pour X = 0 To Largeur - 1
            // Affectation de la couleur lissée actuelle
            Pixel_Intermediaire[X, Y] = Somme_Totale_Pile / Diviseur_Poids
            
            // Glissement de la pile : on retire le pixel le plus à gauche
            Somme_Totale_Pile - Somme_Moitie_Sortante
            
            // On fait évoluer les blocs de la pile
            Somme_Moitie_Sortante - Valeur_Pixel_Sorti_De_La_Pile
            
            // On intègre le nouveau pixel entrant à droite
            Somme_Moitie_Entrante + Valeur_Pixel_Nouveau
            Somme_Totale_Pile + Somme_Moitie_Entrante
            
            Mettre_A_Jour_Pointeurs_Pile()
        Next
    Fin Pour

    // --- PASSE 2 : BALAYAGE VERTICAL (COLONNE PAR COLONNE) ---
    Pour chaque colonne X de l'image
        Initialiser_Pile_Avec_Pixel_Du_Haut()
        Calculer_Sommes_Initiales_De_La_Pile()
        
        Pour Y = 0 To Hauteur - 1
            // Réassemblage final des canaux ARGB sur l'image cible
            Pixel_Cible[X, Y] = Somme_Totale_Pile_Verticale / Diviseur_Poids
            
            // Glissement vertical de la pile vers le bas
            Somme_Totale_Pile_Verticale - Somme_Moitie_Sortante_Verticale
            ...
            Mettre_A_Jour_Pointeurs_Pile_Verticale()
        Next
    Fin Pour
