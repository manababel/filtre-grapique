Analyse d'Image

Documentation - Algorithme de Roberts

Analyse de la structure géométrique par calcul des gradients croisés

1. Pourquoi utiliser le filtre de Roberts ?

  • Extrêmement rapide (Masque 2x2) : L'opérateur de Roberts est l'un des détecteurs de contours les plus légers en calcul. En travaillant sur un voisinage restreint de seulement 2x2 pixels, il nécessite très peu d'accès mémoire et d'opérations mathématiques, ce qui le rend idéal pour les systèmes à ressources limitées.
  • Contours très fins : Grâce à sa petite taille de masque, le filtre calcule un gradient diagonal très localisé. Il génère ainsi des lignes de contours nettement plus fines et précises qu'un opérateur 3x3 (comme Sobel ou Prewitt), bien qu'il soit plus sensible au bruit.
  • Analyse des gradients diagonaux : Contrairement aux filtres traditionnels qui calculent les variations horizontales et verticales, Roberts utilise des différences croisées (diagonales). Combiné au mode d'orientation HSV, il offre une excellente cartographie visuelle des angles obliques dans l'image.

2. Exemples d'implémentation (PureBasic)

Option A : Utilisation Directe (Fonction Standard)

Idéal pour les appels simples et rapides avec passage de paramètres en arguments.

IncludeFile "filtres.pbi"
UseModule filtres

If OpenWindow(0, 0, 0, 800, 600, "Exemple Roberts...", #PB_Window_SystemMenu | #PB_Window_ScreenCentered)
  
  SourceImage = 1
  CibleImage  = 2
  AucunMasque = 0
  Multiply    = 10  ; Multiplicateur de luminance du gradient
  Math        = 0   ; 0: SQR (Euclidien), 1: ABS (Approximation rapide)
  NiveauGris  = 0   ; Convertit le résultat final en niveaux de gris
  Inverser    = 0   ; 1 pour inverser les couleurs (contours sombres sur fond blanc)
  Seuillage   = 0   ; Seuil de coupure (0 = Désactivé, 1-255)
  Orientation = 0   ; 1 pour activer la coloration selon l'angle (HSV)
  AngleAjust  = 0   ; Angle d'ajustement pour le mode orientation (0-360)
 
  file$ = OpenFileRequester("Image","","",0)
  If load_image_32(SourceImage , file$) = 0
    End
  Else
    ResizeImage(SourceImage, 800, 600)
  EndIf
  
  CopyImage(SourceImage,CibleImage)

  ; Appel du filtre de Roberts
  Roberts(SourceImage, CibleImage, AucunMasque, Multiply, Math, NiveauGris, Inverser, Seuillage, Orientation, AngleAjust)
  
  StartDrawing(WindowOutput(0))
  DrawImage(ImageID(CibleImage) ,  0 , 0)
  StopDrawing()
  
  Repeat
    Event = WaitWindowEvent()
    Select Event 
      Case #PB_Event_Gadget
        Select EventGadget()
          Case 1 
            CloseWindow(0)
            End  
        EndSelect
    EndSelect

  Until Event = #PB_Event_CloseWindow
EndIf

Idéal pour la gestion de pointeurs mémoires complexes, les buffers personnalisés ou le traitement par l'API multi-thread en conservant le contexte du filtre.

  FilterCtx\option[3] = 0   ; Inverser (Boolean)

3. Le Filtre : Comment ça marche ?

Le filtre de Roberts est un détecteur de contours minimaliste et ultra-rapide. Son objectif fondamental est de mettre en lumière les lignes et les séparations structurelles d'une image en observant les variations de lumière à une échelle très locale.

Contrairement aux autres filtres qui étudient de larges zones, le filtre de Roberts possède une spécificité mathématique et visuelle unique : il travaille sur un format miniature et se concentre uniquement sur les diagonales.

La force du format 2x2

Pour analyser l'image, l'algorithme regarde un micro-carré composé de seulement 4 pixels (une grille de 2x2). Grâce à cette petite taille, le filtre est capable de détecter des détails extrêmement fins et produit des lignes de contours beaucoup plus minces et serrées que la plupart des autres méthodes.

Le principe des Différences Croisées

Au lieu de mesurer les écarts de luminosité de gauche à droite ou de haut en bas, l'opérateur de Roberts calcule les contrastes en croix. Il effectue deux soustractions en diagonale pour capter les changements obliques :

Première Diagonale (Prendre la pente)
+10
0-1

La règle mathématique visuelle :

Ligne_Détectée = Intensité_Diagonale_1 + Intensité_Diagonale_2

Le filtre compare le pixel supérieur gauche au pixel inférieur droit (+1 vs -1), puis fait l'inverse pour la seconde diagonale. Si un changement brutal de couleur traverse cette croix, le filtre sait qu'un morceau de contour est présent.

Seconde Diagonale (Croiser le calcul)
0+1
-10

Logique de rendu globale (Pseudo-code)

Voici le cheminement logique simplifié que suit l'algorithme sur l'ensemble de la surface graphique :

Pour chaque ligne de l'image
    Pour chaque pixel de cette ligne
        
        // 1. Isoler le carré de 2x2 pixels à partir de la position actuelle
        Pixel_Haut_Gauche = RécupérerPixel(x, y)
        Pixel_Haut_Droit  = RécupérerPixel(x + 1, y)
        Pixel_Bas_Gauche  = RécupérerPixel(x, y + 1)
        Pixel_Bas_Droit   = RécupérerPixel(x + 1, y + 1)
        
        // 2. Calculer l'écart en croix (les deux diagonales)
        Ecart_Diagonale_1 = Pixel_Haut_Gauche - Pixel_Bas_Droit
        Ecart_Diagonale_2 = Pixel_Haut_Droit - Pixel_Bas_Gauche
        
        // 3. Combiner les deux résultats pour obtenir la force totale du contour
        Force_Contour = Combiner_Gradients(Ecart_Diagonale_1, Ecart_Diagonale_2)
        
        // 4. Dessiner le tracé sur la feuille blanche finale
        Dessiner_Pixel_Contours(x, y, Force_Contour)
        
    Next
Fin Pour
